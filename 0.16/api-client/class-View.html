changeState (inherited from StatefulPart)

changeState(state, [options]) : Boolean

Change part's state. If part supports states this method SHOULD be called in response of user interaction (e.g in commands). Also it's called by hosting Region/Area to apply a new state (as result of user navigation request). Implementations SHOULD NOT replace this method, instead they MAY implement onStateChanged.

Parameters:

Name Type Default Attributes Description
state ? Part's state to be applied. State object can have arbitrary structure.
state.name String optional
state.title String optional
options Object optional
options.disablePushState Boolean If true there will be no "statechange" event generated,

i.e. no 'pushstate' done (in terms of AppStateManager). It's being used when AppStateManager applies a new state. |

Returns:

true if event "statechange" was triggered, otherwise false Type: Boolean
