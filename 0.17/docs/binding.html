binding

Контракт

Под binding'ом понимается механизм взаимодействия между двумя объектами, реализующими определенный "интерфейс". Объекты, реализующие его, будем называть bindable значениями. Чтобы стать bindable значением объект должен содержать следующие методы:

get()
set(value)
onchage(handler)

Метод get() возвращает значение, содержащееся в объекте. Метод set(v) - устанавливает значение. Метод onchage(handler) подписывает обработчик handler на изменение значения bindable объекта; может возвращать объект с методом dispose(), в котором обработчик должен отписываться. Метод dispose() будет вызван для разрыва байндинга. 

Любой из этих методов может отсутствовать. Например, если нет метода onchage, то изменения значения bindable объекта не будут отслеживаться, и синхронизация значений двух объектов будет выполнена только один раз. Если нет метода set, то binding будет односторонним.

binding.databind

Для создания связи между двумя bindable значениями используется метод databind объекта binding, возвращаемого из модуля lib/binding.js. 

function (target, source, options) { ... }

, где target и source - это bindable объекты. Метод может так же принимать третий параметр - options, который определяет, каким образом будет построен binding:

Имя поля Описание
oneway Если значением является truthy значение, то независимо от наличия методов target.get и source.set синхронизация будет односторонней - из source в target.

Для поддержания синхронизации в одну сторону - из source в target объект source должен содержать метод get() и onchage(handler), а объект target - метод set(v). Дальше все просто - если в source меняется значение, то вызывается метод source.onchage, который сигнализирует объекту binding забрать значение из source (source.get) и поместить его в target (target.set).

var error;
try {
    target.set(source.get());
} catch (ex) {
   error = ex;
}

Если объект source реализует метод setError, то в него будет передана ошибка синхронизации. С обратным binding'ом те же правила. Причем второй раз вызывать binding.databind не нужно - просто, если объект target реализует методы нужные для того, чтобы быть источником, а объект source - методы, присущие получателю, то обратный binding будет поднять автоматически.

binding.html

Часто возникает необходимость синхронизации значения html элемента с свойством доменного объекта. Для этого необходимо, чтобы html-элемент был обернут в bindable значение. Такую функциональность реализует метод bindable.html.

function (el, options)
{ ... }

, где el - html элемент или jQuery множество. Объект options описан ниже:

Имя поля Описание
getSpecific() Метод может возвращать объект, которым будет расширен объект options. -Целесообразность включения этого метода в options при подаче параметром в метод html() не очевидна.-
accessor() Реализация доступа к свойству html-элемента. Фактически та самая "обертка".
event Имена событий html-элемента через пробел, по которым должен вызываться метод onchange результирующего bindable значения html элемента.
eventHandler() Если указан, то будет вызываться каждый раз вместе с переданным в onchage обработчиком.
parse() Если указан, то вызывается перед вызовом accessor(v). В результате в accessor идет значение, которое вернул метод parse(v).

|

Однако самостоятельно описывать объект options необязательно, binding.html может определять типовые настройки, исходя из типа html-объекта. Соответственно объект options можно не указывать, тогда он будет создан на основе типа html-элемента. Так же можно в качестве объекта options передать одну из следующих строк:

Метод возвращает bindable-значение.

binding.expr

При помощи данной функции можно создавать bindable-выражения:

language=javascript binding.expr = function (source, expr) { return new BindableExpression(source, expr); };В качестве source указывается контекст, в котором будет выполняться выражение. Следует помнить, что выражение может следить только за своим контекстом (его свойствами) и аргументами, переданными в функцию выражения. binding.expr же ничего не передает в качестве аргументов, поэтому такое BindableExpression будет следить только за своим контекстом, к которому внутри функции можно обратиться через this.

binding.domain

Абсолютно то же самое, что и binding.expr. Используется для обратной совместимости.

binding.commandBind

Связывает html элемент с командой. В результате по клику на элемент, а так же по нажатию пробела или Enter, происходит выполнение команды. При изменении свойства canExecute() команды происходит обновление доступности html-элемента.

Функция возвращает объект с методом dispose, который необходимо вызвать для разрыва binding'а.

Пример

Рассмотрим следующий код, взятый из binding.commandBind:

language=javascript disposable = binding.databind( binding.html(element, "enabled"), binding.domain(cmd, "canExecute") );Вызов binding.html(element, "enabled") создает bindable значение, accessor которого будет проставлять или снимать признак disabled у элемента. Это значение будет целью (target) синхронизации. Вызов binding.domain(cmd, "canExecute") создает bindable-выражение, которое возвращает значение свойства cmd.canExecute(). Это выражение, являясь само по себе bindable значением, будет источником (source). Причем при изменении значения свойства canExecute команды будет вызван метод onchage выражения, что приведет к пересинхронизации значений.

Обратного binding'а создано не будет, т.к. параметр target метода binding.databind не содержит onchange.
