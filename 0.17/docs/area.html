Area

Область, Area, это объект-контейнер, связанный с DOM-элементом. Используется для переключения состояния приложения. В приложении может быть несколько Area, но только одна из них является активной, т.е. видимой. Объекты Areas управляются и хранятся в AreaManager. Область имеет множество состояний - AreaState и регионов - Region.

Определение

Обычно области определяются в HTML разметке с помощью css-класса x-area. Экземпляр Application при старте (Application.run) находит все элементы с классом x-area и создает связанные с ними объекты Area. Объекты Area добавляются в AreaManager, единственный экземпляр которого хранится в Application (поле Application.areaManager). Для объекта Area помимо DOM-элемента может быть задано имя. Имя области берется из data-атрибута area, либо при его отсутствии, из атрибута id. Одна из областей может не иметь имени. Примеры определения областей:

<div class="x-area" data-area="area1">
<div class="x-area" id="area2" style="display: none; ">
<div class="x-area" style="display: none; ">

Если HTML-разметка не содержит элементов с классом x-area, то автоматически создается одна неименованная Area, связанная с document.body.

Область по умолчанию

Одна из областей может быть областью по умолчанию. Это та область, которая будет активирована при переходе в корень сайта. Областью по умолчанию является:

  • единственная область
  • неименованная область
  • область с опцией isDefault:
<div class="content x-area" data-area-options="isDefault:true">

API

Получение доступа к объекту Area

Получить доступ к экземпляру Area можно следующим образом:

  • используя экземпляр Application
var app = Application.current;
var area = app.areaManager.getArea("myarea")
  • либо определив Area-модуль (в этом случае callback-функция будет вызвана при первой активации области):
define(["core"],
function (core) {
  "use strict";
  // "myarea" - имя области так, как оно описано в разметке
  core.createAreaModule("myarea", function (app, area) {
    var region1 = area.regionManager.getRegion("myregion");

  });
});

Подробней про App-модули.

Управление состояниями

Для добавления состояния (AreaState) используется метод addState:

 * Add a new state into the area.
 * @param {Object|String} stateSpec - name of the state or json object with its specification
 * @param {Object} map An object which keys are regions names and values are parts (names or instances)
 */
addState:function (stateSpec, map) {

В качестве map передается объект-словарь, где полями являются наименования регионов области, а их значениями имена (или экземпляры) партов для этих регионов. Пример:

area.addState({name:"состояние1", title:"заголовок_состояния_для_UI"}, {"регион1" : "парт_для_региона1"});

При активации данного состояния ("состояние1"), в регионе с именем "регион1" будет активирован парт "партдлярегиона1".

При активации Area по умолчанию активируется состояние по умолчанию. Его можно задать с помощью метода setDefaultState:

area.setDefaultState("состояние1");

Управление областью

Для переключения активной области следует использовать API AreaManager'a:

app.areaManager.activateArea(area);

Но помимо явного API, активация областей происходит через клиентскую навигацию.

Меню состояний области

Для создания меню, пункты которого соответствуют состояниями Area (определенным ранее) предназначен класс AreaStatesMenu (наследующий от Menu). Т.е. AreaStatesMenu - это просто меню как структура, для конкретного отображения в UI следует задать презентер. Пример:

core.createAreaModule("myarea", function (app, area) {
    var menuRegion = area.regionManager.getRegion("mainmenu");
    // наполнить area состояниями
    var menu = new AreaStatesMenu(area);
    var presenter = MenuPresenter.create({viewModel: menu, orientation: "horizontal"});
    menuRegion.activatePart(presenter);
});
