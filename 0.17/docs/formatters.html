Formatters

Значения свойств для отображения в UI-компонентах не могут использоваться as is, необходимо дополнительно форматирование. В частности для дат, перечисляемых свойств и объектных свойств должно быть более понятное пользователю представление, чем результат стандартного метода Object.toString. Форматирование значений свойств (и объектов) выполняют т.н. форматеры (formatters).

Метаданные

Для свойства в метаданных может быть задана функция formatter:

var metadata = {
    entities: {
        Survey: {
            props: [
                title: { vt: "string", formatter: function(v) { return v; }},
                . . .
            ]
        }
    }
};

Если функция не задана, то при инициализации модели устанавливается дефолтный форматер свойства на основании его типа. Для этого используется объект-модуль formatters ("lib/formatters"). У него получается форматер по метаописанию свойства:

if (!propInfo.formatter) {
        var formatter = formatters.getFormatter(propInfo);
        if (formatter) {
            propInfo.formatter = (function(formatter) {
                return function(value) {
                    return formatter(this, value);
                }
            })(formatter);
        }
    }

Стандартные форматеры существуют для свойств типов: date, time, dateTime, enum. Для enum'ов форматер возвращает описательные имена элементов перечислений. Некоторые стандартные форматеры свойств поддерживают задание формата в метаданных свойства - атрибут format (т.е. format - это фасет):

{ name: "timestamp", vt: "dateTime", format: "YYYY.DD.MM HH:mm:SS" }

Сейчас декларативный формат поддерживается для свойств типов date, time, dateTime. Значение формата - формат-строка для форматирования объекта Date с помощью библиотеки moment (см. xfwajax:Libraries).

Runtime

В рантайме для форматирования значения могут использоваться две функции (помимо вызова форматера напрямую):

  • DomainObject.getFormatted:
/**
 * Return formatted property value
 * @param {String} propName property name
 * @return {String} formatted value
 */
getFormatted: function(propName) {
  • formatters.formatPropValue:
/**
 * Format arbitrary object's property value.
 * @param {String} propMeta property metadata
 * @param {*} propValue property value to format
 * @return {String} formatted property value
 */
formatPropValue: function (propMeta, propValue) {}

Первая функция используется для получения форматированного значения для заданного свойства, вторая - когда значение уже есть и его надо отформатировать.

Форматирование значений свойств используется в списке (ObjectList/SlickObjectListPresenter) и редакторах свойств (peObjectList/peObject). Помимо этого, для списка/редакторов свойств, форматеры можно задать явно. Для списка форматер задается как функция в поле formatter метаописания колонки:

var list = core.ui.ObjectList.create(app, {
                uow: options.uow,
                dataSource: options.dataSource,
                autoLoad: true,
                columns: [
                    { title: 'Создан', name: 'created' },
                    { title: 'Тема', name: 'title' },
                    {
                        title: 'Организатор',
                        name: 'organizer',
                        formatter: function (value) {
                            var firstName,
                                lastName;
                            if (value && value.isLoaded) {
                                firstName = value.firstName();
                                lastName = value.lastName();
                                return (firstName ? firstName + " " : "") + lastName;
                            }
                        }
                    }
                ],
                menuList: {
                    off: ["Create"]
                },
                orderby: ["-created", "title"],
                filter: "ObjectFilter:surveys"
            });

Для редактора свойства форматер задается в метаданных редактора свойства (которые является расширением метаданных свойства):

app.registerPart("ObjectEditor:Survey", function (options) {
    var editor;
    options = options || {};
    editor = core.ui.ObjectEditor.create({
        title: "Редактор опроса",
        pages:[
            {
            name:"main",
            title:"Основные",
            properties:[
                "title",
                { name:"text", rows:5 },
                {
                    name: "organizer",
                    // форматер как имя форматера, заданного для класса
                    formatterName: "login",
                    // ИЛИ как функция:
                    formatter : function(user) {
                        return user
                            ? user.firstName() + " " + user.lastName() + " (" + user.login() + "
                            : null ;
                    }
                }, // skipped

С помощью атрибута formatterName задается имя форматера, определенного в классе, см далее.

Форматирование навигируемых скалярных свойств в peObject

Для навигируемых свойств по умолчанию нет форматтера (хотя он может быть задан явно в метаданных свойства). Методы getFormatted/formatPropValue в этом случае возвращают результат метода toString, вызванного для значения свойства. А т.к. значение навигируемого свойства - DomainObject, то вызывается DomainObject.toString. Дополнительно, метод formatPropValue передает атрибут formatterName метаописания свойства в вызываемый метод toString.

Метод DomainObject.toString, если вызван без параметров, использует дефолтный форматер для типа (берется объект formatters.default из метаописания типа). Если передан параметр, то ожидается что это наименование форматера из коллекции форматеров типа - см. далее.

Форматеры типов

В метаданных типа (EntityType) может быть задан атрибут formatters, в нем должен содержаться объект, полями которого являются наименования форматеров, а их значениями функции-форматеры:

metadata.entities.User.formatters = {
        "default": function() {
            return this.firstName() + " " + this.lastName();
        },
        login: function() {
            return "Логин: " + this.login();
        }
    }

Значением this в функциях-форматерах является текущий доменный объект. При инициализации модели, если для типа не задан дефолтный форматер (formatters.default), то он устанавливается по следующему алгоритму:

  • если у типа есть строковое ненулабельное свойство, то дефолтный форматер возвращает его значение
  • иначе, возвращается строка в виде: 
    {Описание типа} ({Наименование типа}) (Id: {Идентификатор объекта})
