Getting Started Tutorial

Практическое руководство для быстрого начала разработки на WebClient с помощью NuGet-пакета WebClient.Sample

Протестировано для версии 0.13. Для предыдущих версий: 0.12.

Готовый репозиторий проекта:

git clone http://rnddev04.lab.croc.ru:9000/projects/RND/repos/webclient-tutorial-simple

{toc}

Начало работы

Для начала разработки приложения на WebClient нам потребуется:

  • установленная Visual Studio 2013/2012 (туториал на примере VS2013)
  • настроенный репозиторий nuget-пакетов croc r&d, подробно написано здесь - xfw3:Release Management. Если кратко, то в настройках NuGet Package Manager (раздел Options/Package Manager/Package Sources) добавить источник http://nuget.rnd.croc.ru/nuget.

Создание проекта

В Visual Studio выполним следующее:

  • создадим новый проект, выбрав шаблон ASP.NET Web Application, в качестве наименования выберем Milestones
  • в открывшемся диалоге выберем шаблон Empty
  • в свойствах созданного проекта изменим корневой неймспейс и имя сборки на Croc.Samples.Milestones
  • в Solution Explorer'e откроем контекстное меню на проекте и выберем пункт "Manage NuGet packages.."
  • в открывшемся диалоге "Manage NuGet Packages" перейдем на закладку Online и выберем источник пакетов "CROC R&D" (http://nuget.rnd.croc.ru/nuget) (в выпадающем списке можно выбрать "Include Prerelease" чтобы установить SNAPSHOT-пакеты разрабатываемой версии - но обратите внимание, что руководство может не соответствовать dev-версии, поэтому рекомендуется использовать релизные пакеты)
  • установим пакеты:
    • CROC XFW3 WebClient Sample
    • CROC XFW3 Storage Provider for Microsoft SQL Server (или любой другой Storage Provider)
  • если у вас не установлен XFW3 DslDesigner, то его необходимо установить. Для этого надо либо установить nuget-пакет CROC XFW3 DslDesigner (после установки пакета запустится установка расширения для VS - DslDesigner'a, если инсталлятор не запустился, выполните файл Croc.XFW3.DslDesigner.vs2013.vsix вручную из папки tools пакета), либо установить дизайнер из кастомной галереи http://dev.rnd.croc.ru/vsgallery (подробней).

Обратите внимание, что при установке пакета Sample используется текущее имя сборки и корневой неймспейс проекта (они подставляются в app.config и некоторые генерируемые cs-файлы). Поэтому, если вы измените имя сборки и/или неймспейс проекта после установки nuget-пакета, то необходимо будет скорректировать имена неймспейсов/типов в различных местах. Как минимум в app.config имя сборки используется для указания типа фабрики доменных объектов:

<implementation factory-provider=" Croc.Samples.Milestones.DomainObjects.Factories.FactoryProvider,  Croc.Samples.Milestones" />

Подключение XFW3

Клиентская часть WebClient может работать без XFW3-сервера и вообще без .NET-сервера, однако типовым использованием является конечно XFW3 на сервере. При установке пакета XFW3.WebClient.Sample в проект добавляется файл app.config, содержащий шаблон конфигурации XFW3-приложения. Также выполняется трансформация файла web.config.

В созданном файле app.config необходимо изменить конфигурацию подсистемы Storage, задав правильные строки соединения. Далее необходимо создать БД. Скрипт для создания пользователей БД можно найти в папке Tools пакета XFW3.Storage.MsSql (примерно: packages\XFW3.Storage.MsSql.1.13.0\Tools\create-users.mssql2005.sql).

Обратите внимание, что скрипт и строки соединений в app.config подразумевают, что для SQL Server включена SQL Server Authentication

Доменная модель и артефакты

Для запуска XFW3-приложения необходимо создать доменную модель. Подробное описание моделирования можно найти в Учебном курсе. Сейчас же будем исходить из того, что читатель знаком с основами моделирования для XFW3. Для простоты к статье приладается xdm-файл доменной модели . {attachments:patterns=domainModel.xdm}

Сохраните этот файл в подпапке VS-проекта (например "Domain Model"), т.к. все пути в xdm-файле начинаются с "..". Также обратите внимание, что у сохраняемого файла модели правильное расширение - .xdm (IE может его изменить)

Открыв доменную модель, сгенерируем артефакты:

  • xml-метаданные - в Metadata\metadata.xml
  • C#-классы доменных объектов - в DomainObjects\GeneratedDomainObjects.cs, неймспейс - Croc.Samples.Milestones.DomainObjects
  • SQL-DDL скрипт создания структуры БД (расположение непринципиально)
  • JavaScript доменную модель - в client\app\domain\model.js (а также надо сгенерировать "модуль расширения доменной модели" в client\app\domain\model-ext.js - это надо сделать один раз)

    Обратите внимание, что js-модель генерируется в отдельной каталог domain - его наименование используется в ссылках в app.js на модуль модели model-ext.js. После генерации в папке domain должна также появится папка с ресурсами - nls

Убедимся, что сгенерированные файлы попали в состав проекта. Если при генерации была проставлена галка "Генерировать файл ресурсов", то из подкаталогов каталога client/app/nls нужно удалить файлы resources.js.bak. В противном случае будут проблемы с включением offline-режима приложения.

Пути и файлы, упомянутые выше используются в файле app.config, который добавляется в проект при установке пакета WebClient.Sample. Если вы выберите другое расположение файлов, необходимо будет скорректировать их пути в app.config

Далее необхдимо создать БД и скорректировать строки соединения в app.config.

Первый запуск

После того, как мы сгенерировали все артефакты, создали структуру БД и пользователей, корректно задали параметры в app.config (имена классов, пути файлов, строки соединения), можно запустить приложение (F5). При запуске браузера должна открыться страница корня сайта, практически без содержания. Это говорит нам о том, что во View и startup-коде веб-приложения нет ошибок. Запускать приложение рекомендуется в Google Chrome (Tip: как установить Chrome в офлайне). После запуска сайта без ошибок следует активировать Developer tools (F12) и перейти на закладку Console. Там не должно быть никаких ошибок.

Далее создадим новый js-файл - client\app\module-main.js и подключим его в client\main.js (обратите внимание, что модуль импортируется без расширения):

require([
    "jquery",
    "core",
    ...
    "app/module-main" // <<<<<<<<<<<<<<<<<<
], function ($, core, model, DataFacade, BackendInterop, NavMenu) {
    ...
});

В файле module-main.js мы опишем Area-модуль - это функция которая будет вызвана при первой активации Area.

define([
    "core",
    "lib/ui/menu/AreaStatesMenu",
    "lib/ui/menu/MenuNavPresenter"
], function (core, AreaStatesMenu, MenuPresenter) {
    "use strict";

    core.createAreaModule(function (app, area) {
        var menuRegion = area.regionManager.getRegion("mainmenu"),
            menu,
            presenter;

        core.lang.forEach(app.model.meta.entities, function(typeInfo, name) {
            if (!typeInfo.temp && !typeInfo.abstract) {
                area.addState({ name: typeInfo.name, title: typeInfo.descr }, { "main": "ObjectList:" + typeInfo.name });
            }
        });

        menu = new AreaStatesMenu(area);
        presenter = MenuPresenter.create({ viewModel: menu, radio: true, orientation: "vertical" });
        menuRegion.activatePart(presenter);

        area.setDefaultState("Project");
    });
});

Здесь мы создали меню, пунктами которого являются команды активации парта списка объектов (ObjectList:имя_типа). Меню мы установили в качестве парта региона mainmenu. При вызове команд активация будет происходит в регионе main. Но по умолчанию в регионе будет отображаться список объектов Project. Это мы задали с помощью состояния области - для состояния ("default") мы определили отображение региона ("main") на парт ("ObjectList:Project"):

area.addState("default", { "main": "ObjectList:Project" });

Раскоментируем запуск приложения в файле main.js:

$(document).ready(function () {
        setTimeout(function () {
            app.run();
        }, 0);

    });

Запустив приложение мы увидим дефолтное приложение с меню для каждого типа и списком объектов. Из списка открывается создание и редактирование объектов.

Модули

В main.js подключаются так называемые app модули (файлы из папки modules). Это модули для расширения функциональности приложения, не являющиеся обязательными для всех приложений. Шаблон main.js, добавляемый в проект после установки nuget-пакета XFW3.WebClient.Sample содержит некоторое количество модулей. Мы можете изменить их набор при необходимости. Для данного демо приложения рекомендуется следующий набор:

"modules/affix/module-affix",
    "modules/breadcrumbs/module-breadcrumbs",
    "modules/debug/module-debug",
    "modules/drafts/module-drafts",
    "modules/notifications/module-notifications",
    "modules/scroll-accelerators/module-scroll-accelerators",
