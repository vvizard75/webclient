Icons

Для пиктограмм (иконок) в WebClient используется векторный шрифт. Шрифт поставляется в дистрибутиве в папке src/content/fonts и представляет собой множество файлов (eot, svg, ttf, woff). Наличие множества файлов вызывано тем, что разные бразуеры поддерживают разные форматы.

Поддержка браузерами

Для вставки пиктограммы в Html используются псевдо-классы (":before"). Они поддерживаются всеми бразурами кроме IE6/7.

Для кастомного шрифта используется CSS3 @font-face. font-face поддерживается практически всеми браузерами (подробней см. http://caniuse.com/#feat=fontface).

Но форматы файлов шрифтов, используемые в font-face в разные браузерах поддерживаются разные:

Format Support
WOFF - Web Open Font Format Desktop: IE9, Chrome, FF, Safari 5.1, Opera 11.1

Mobile: iOS 5.0, Android 4.4, IEMobile 10, Android Chrome 30, Android FF 25, BB 7 | | EOT - Embedded OpenType fonts | IE6 | | TTF/OTF - TrueType and OpenType font support | Desktop: FF, Chrome, Safari 4.1, Opera 10

Mobile: iOS 4.2, Android 2.2, Android Chrome 30, Android FF 25, BB 7 | | SVG | Desktop: Chrome, Safari 5.1, Opera 9

Mobile: iOS 3.2, Android 3.0, BB, Android Chrome 30 |

Подробней см. http://caniuse.com/#feat=fontface

Итого получается, что для большинства браузеров подходит формат WOFF. Кроме старых IE (до 9), iOS до 5 и Android до 4.4. Для IE до 9 подходит формат EOT. Для Android и iOS подходит TTF/OTF, кроме совсем старых iOS (до 4.1). Для iOS младше 4.1 подходит SVG.

Все это различие в форматах скрыто в определении @font-face в icons.css

HTML

В icons.css для каждого символа шрифта есть css-класс, начинающийся на "x-icon-". Классы иконок используются следующим образом:

<a href="#"><span class="x-icon-edit" aria-hidden="true"></span>Редактировать</a>

Атрибут aria-hidden опциональный, нужен только для VoiceOver на OS X (подробнее). Для классов x-icon-* добавлен небольшой margin справа, чтобы иконка не сливалась с текстом. Таким образом, не нужно писать лишний   (пробел) или добавлять margin руками. Однако, если иконка отображается без сопроводительного текста, то margin может быть лишним. В этом случае нужно добавить класс x-icon-alone, убирающий правый margin.

Редактирование шрифта

Шрифт содержит типовой набор символов, используемых компонентами WebClient. В прикладном проекте может потребоваться расширить состав символов или изменить его. Сделать это можно несколькими способами:

  • переопределить CSS-классы
  • заменить символы в шрифте

В обоих случаях потребуется изменить шрифт (в первом случае чтобы добавить символы, во-втором чтобы изменить).

Помимо этого в прикладном проекте может потребоваться добавить новые символы. Это можно сделать как введя новый шрифт и классы для него, так и расширив существующий.

В дистрибутив клиента входит файл проекта IcoMoon (Design\icomoon\selection.json), используя который можно редактировать векторный шрифт в приложении IcoMoon (Chrome extension) или на сайте http://icomoon.io.

При редактировании рекомендуется добавлять новые символы с Unicode-кодами последовательно после последнего ("\e027" => "\e028" и т.д.) При расширении существующего шрифта (icomoon) надо будет только добавить новый css-класс с кодом символа:

.x-icon-my-fancy-symbol:before {
    content: "\e027"; /*my-fancy-symbol*/
}

У каждого символа в шрифте есть имя. Имена даются так, чтобы описывать само изобращение. Тогда как css-классы называются в соответствии с use-case'ом. Например, символ pencil используется для класса x-icon-edit.

IconProvider

Для задания иконок каким-то элементам в javascript-коде можно вручную формировать необходимый html с использованием нужного css-класса. Например, для пункта меню:

var menu = { items: [
    {
        name: "Edit",
        title: "Редактировать",
        html: "<span class='x-icon-edit' aria-hidden='true'></span>Редактировать"
    }
]};

Однако, это громоздко и неудобно. Чтобы упростить формирование html для иконок используется компонент IconProvider. От реализации требуется лишь наличие метода getIcon, принимающего наименование и возвращающего html, соответствующий иконке. В рамках WebClient поставляется единственная реализация IconProvider в файле lib/ui/IconProvider. В данном классе прописано отображение имен всех стандартных действий на соответствующие css-классы иконок, метод getIcon формирует html аналогичный приведенному в примере выше. Т.о. можно облегчить описание меню:

var menu = { items: [
    {
        name: "Edit",
        title: "Редактировать",
        html: iconProvider.getIcon("edit") + "Редактировать"
    }
]};

Но в случае меню, можно еще упростить описание:

var menu = { items: [
    {
        name: "Edit",
        title: "Редактировать",
        icon: "edit"
    }
]};

Дело в том, что за формирование html для menuItem отвечает статический метод Menu.getItemHtml, который используется всеми презентерами меню. Этот метод использует статический экземпляр IconProvider, который должен быть доступен через поле core.ui.iconProvider. Ответственность за заполнение этого поля возложена на прикладное приложение.

Best practice: определять отдельный AMD-модуль app/iconProvider, в котором формировать экземпляр IconProvider и заполнять поле core.ui.iconProvider. При необходимости там же iconProvider может быть расширен для поддержки иконок для кастомных действий.

Для меню также поддерживается отображение имени пункта на иконку по умолчанию: если имя пункта меню совпадает с именем иконки, поддерживаемой core.ui.iconProvider, то эта иконка автоматически используется для данного пункта. При этом регистр первой буквы наименования может быть как нижним, так и верхним. Т.о. в предыдущем примере можно вообще опустить поле icon, соответствующая иконка магическим образом подставится в html.

Статическое поле core.ui.iconProvider позволяет получать иконки по именам практически отовсюду. Так, для удобства использования иконок в Handlebars-шалбонах существует хелпер icon, который принимает имя иконки в атрибуте name. Пример:

<a href="#">{{icon name="edit"}}Изменить</a>
