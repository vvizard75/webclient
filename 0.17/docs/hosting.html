Hosting

На текущий момент в качестве хоста для WebClient приложений поддерживается только IIS (т.к. ASP.NET MVC использует System.Web).

Для корректного функционирования приложения неоходимо выполнить некоторые настройки в web.config для IIS и ASP.NET.

Большинство настроек устанавливаются в нужные значение при установке NuGet пакета WebClient.Server

Настройки ASP.NET выполняются в секции system.web, настройки IIS - в system.webServer

requestPathInvalidCharacters

URL'ы страниц в WebClient могут содержать знак двоеточия ":". Например: http://localhost/ajax/display/admin/users/ObjectEditor:User?type=User&id=59ce451a-65ba-480b-8853-4f13924c53d3 "ObjectEditor:User" - это имя парта. По умолчанию знак двоеточия в ASP.NET запрещен. Без дополнительных настроек при старте сервера будет возникать HttpException:

A potentially dangerous Request.Path value was detected from the client (:)

Чтобы этого избежать необходимо в задать атрибут httpRuntime/requestPathInvalidCharacters:

<httpRuntime requestPathInvalidCharacters="" />

По умолчанию, значение requestPathInvalidCharacters (т.е. запрещенные символы): "<,>,*,%,&,:,\" (см. MSDN).

Максимальный размер запроса

По умолчания IIS и ASP.NET ограничивают максимальный размер запроса. Это важно при загрузке файлов (в качестве значения binary-свойств). Ограничение ASP.NET задается в system.web/httpRuntime/@maxRequestLength. Ограничение IIS - в system.webServer/security/requestFiltering/requestLimits/@maxAllowedContentLength. Эффективное ограничение - это минимум из двух значений! Значение maxAllowedContentLength задается в байтах, maxRequestLength в киллобайтах.

| Ограничение | Тип | Значение по умолчанию | Максимальное значение | | IIS (maxAllowedContentLength) | UInt32 | 30000000 (bytes, ~28.6MB) | 4294967295 (4Gb) | | ASP.NET (maxRequestLength) | Int32 | 4096 (KB, 4Mb)| 2147483647 (2Tb) |

Таким образом теоретический максимальный размер файла, который может быть загружен, 4Gb. Однако, по факту файлы больше 2Gb не загружаются. Подробней см. FileController.

Чтобы обойти это ограничение FileController и peBinary поддерживают кусочную загрузку (chunked upload)

регистрация MIME-типов

Для корректной загрузки файлов шрифтов и шаблонов, в секции system.webServer необходимо определить расширения и их mime-типы. По умолчанию в machine.config не для всех файлов шрифтов прописы mime-типы, поэтому IIS не считает их статическим содержимым.

<staticContent>
            <remove fileExtension=".md" />
            <mimeMap fileExtension=".md" mimeType="text/plain" />
            <remove fileExtension=".appcache" />
            <mimeMap fileExtension=".appcache" mimeType="text/cache-manifest" />
            <remove fileExtension=".ttf" />
            <mimeMap fileExtension=".ttf" mimeType="font/ttf" />
            <remove fileExtension=".eot" />
            <mimeMap fileExtension=".eot" mimeType="application/vnd.ms-fontobject" />
            <remove fileExtension=".otf" />
            <mimeMap fileExtension=".otf" mimeType="font/otf" />
            <remove fileExtension=".woff" />
            <mimeMap fileExtension=".woff" mimeType="application/x-font-woff" />
            <remove fileExtension=".svg" />
            <mimeMap fileExtension=".svg" mimeType="application/vnd.ms-fontobject" />
            <remove fileExtension=".webapp" />
            <mimeMap fileExtension=".webapp" mimeType="application/x-web-app-manifest+json" />
            <remove fileExtension=".hbs" />
            <mimeMap fileExtension=".hbs" mimeType="text/x-handlebars-template" />
        </staticContent>
