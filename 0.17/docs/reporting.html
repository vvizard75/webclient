Reporting

Общее описание

WebClient содержит модуль отчетов (reporting). Он состоит из серверой и клиентской частей. Серверная часть распространяется как отдельный NuGet-пакет XFW3.WebClient.Reporting, зависящий от XFW3.WebClient.Server и XFW3.ReportService.Server. Клиентская часть поставляется в составе основного дистрибутива в виде модуля module-reporting.

Модуль отчетов предоставляет две основные возможности:

  • построение и отображение отчетов ReportService
  • экспорт данных списков в различные формате

Отчеты представляют собой парты, отображающие html контент, сгенерированный подсистемой RS. Построеные отчеты могут отображаться в рамках приложения, либо в отдельном окне как отдельное SPA приложение. В составе основного приложения отчет отображается с помощью парта core.ui.ReportPart. Так же любой отчет (снару или из парта ReportPart) может быть открыт в отдельном окне браузера. В данном случае используется парт core.ui.ReportPagePart в составе отдельного SPA приложения.

Любой отчет может быть экспортирован в форматы, подключенные в настройках подсистемы Reporting, с помощью RS Renderer'ов. Они же используются для экспорта данных списков.

Далее описана функциоанальность отчетов. Экспорт данных списков см. в статье Экспорт данных списков.

Параметры партов

Общие параметры для core.ui.ReportPart и core.ui.ReportPagePart

Параметр Обязательность, значение по умолчанию Описание
reportName обязателен имя профиля отчета в подсистеме RS
refresh false признак получения нового отчета (см. Кэширование данных отчетов)
dontCacheXslFo false не кэшировать отчет Кэширование данных отчетов
affixTableHeader true заголовки таблиц прилипают к краю visual viewport. функционал реализован с помощью Affix Manager
params undefined параметры отчета в виде json объекта
menu undefined метаданные меню. необязателен.
commands undefined объект, содержащий реализации команд
Presenter undefined функция презентера. если не задан, то по умолчанию используется core.ui.ReportPartPresenter или core.ui.ReportPagePartPresenter
presenter undefined экземпляр презентера. если не задан - используется опция Presenter

Параметры core.ui.ReportPart

Параметр Обязательность, значение по умолчанию Описание
filter undefined экземпляр фильтра или имя парта фильтра
filterExpanded true свернут или развернут фильтр
autoGenerate false генерировать ли отчет сразу после открытия парта
expandFilterTitle resources\["objectFilter.show"\]("objectFilter.show"\.html) текст ссылки для разворачивания фильтра
collapseFilterTitle resources\["objectFilter.hide"\]("objectFilter.hide"\.html) текст ссылки для сворачивания фильтра

Пример регистрации парта отчета

app.registerPart("SurveysReport", function (options) {
        var reportPage = reporting.ReportPart.create(app, core.lang.extend({
            reportName: "Surveys",
            filter: "ObjectFilter:surveysReport"
        }, options));
        return reportPage;
    });

Клиентская обработка ссылок в отчетах

В партах отчетов определённого рода ссылки обрабатываются на клиенте, т.е. самим партом. Подобные ссылки начинаются с префикса (определение протокола в uri) webClient:// и имеют следующий вид:

webClient://Имя действия[/Спецификация действия]?Параметры

Именем действия и спецификациями могут быть:

Имя Спецификация Описание
OpenReport имя отчета в подсистеме RS открывается заданный отчет
Navigate имя парта выполняется навигация в заданный парт
имя кастомной команды не используется выполняется заданная в парте отчета или его параметрах команда

Примеры ссылок:

  • из текущего отчета, например Surveys, откроется отчет Survey (в парте по умолчанию ReportPart:Survey или ReportPagePart:Survey) с установленным параметром survey, сразу после открытия выполнится автогенерация отчета:
webClient://OpenReport/Survey?autoGenerate=true&params.survey=3257e562-a02a-482d-a80d-7393210892bd
  • из текущего отчета будет выполнен переход в парт SomeReportPart, в фильтр парта отчета будут установлены значения ограничений title и organizer
webClient://Navigate/SomeReportPart?autoGenerate=true&params.title=text&params.organizer=3257e562-a02a-482d-a80d-7393210892bd
  • будет выполнена команда CustomCommand с передачей ей параметров param1 и param2
webClient://CustomCommand?param1=fubar&param2=42

Указываемые в виде url encoded query string параметры, после формирования из них json объекта передаются в открываемый парт (в т.ч. парт отчета) как опции создания парта. Т.е. query string вида 

filterExpanded=false&autoGenerate=true&params.reportParam1=val1&params.reportParam2=val2

будут переданы в конструктор парта в виде json объекта

{
    filterExpanded: false,
    autoGenerate: true,
    params: {
        reportParam1: val1,
        reportParam2: val2
    }
}

Действие Navigate не обрабатывается в отчете, открытом в отдельном окне. Клик по такой ссылке не приводит к выполнению каких либо действий.
