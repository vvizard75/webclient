ObservableCollectionView

Класс ObservableCollectionView

Класс ObservableCollectionView представляет собой обертку над коллекцией элементов (простым массивом или ObservableCollection), образующую представление коллекции в заданном пользователем виде. Управлять можно можно двумя параметрами: сортировкой и фильтрацией.

Рассмотрим подробнее методы ObservableCollectionView:

Названиие Сигнатура Описание
constructor function (source) Создание view на основе source, если задано. source может быть массивом или ObservableCollection
dispose function () Отписка внутренних обработчиков от событий и освобождение ресурсов
source function (value) Установка (получение при вызове без параметров) коллекции-источника (параметр аналогичен конструктору).
orderBy function (orderBy) Установка (получение при вызове без параметров) сортировки view. Параметр представляет собой имя свойства (или массив свойств) по которым будет осуществляться сортировка. Наименование свойств можно дополнять служебными символами: +prop или prop - сортировка по возрастанию, -prop по убыванию, *prop - направление сортировки обратное тому, что сейчас задано по этому свойству. Если в данный момент сортировки по этому полю нет (или не совпадает порядок свойств) то по возрастанию.
where function (filter) Установка (получение при вызове без параметров) фильтра view. Из коллекции-источника будут выбраны только элементы, для которых filter вернет true.
all function () Возвращает все элементы view (т.е. все элементы коллекции с учетом orderBy и where).
get function (i) Получает элемент с индексом i. Индексы считаются согласно порядку элементов во view (т.е. с учетом orderBy и where).
count function () Количество элементов во view (т.е. с учетом where)
indexOf function (item) Индекс элемента во view. Индексы считаются согласно порядку элементов во view (т.е. с учетом orderBy и where).
add function (items) Добавление элемента (или массива элементов) в исходную коллекцию.
remove function (items) Удаление элемента (или массива элементов) из исходной коллекции.
move function (indexFrom, indexTo) Элементы с номерами indexFrom и indexTo меняются местами d исходной коллекции. Индексы считаются согласно порядку элементов во view (т.е. с учетом orderBy и where).
clear function () Очистка исходной коллекции
reset function (items) Сброс bc[jlyjq коллекции. Все элементы коллекции удаляются, затем добавляется элемент (или элементы если items массив) items. ](http://wiki.rnd.croc.ru/display/

Важным моментом является то, что вызов любого метода изменения коллекции генерирует только одно событие. Т.е. вызов метода reset не вызовет событие удаления для каждого старого элемента, а потом добавление для каждого нового, а только одно событие изменения коллекции.

С точки зрения производительности, важно отметить, что ObservableCollectionView оптимизирует количество вычислений внутри себя. Фактически применение where и orderBy происходит, подобно LINQ, только в момент обращения. После вычисления "представления" view кэширует его внутри, и не вычисляет при повторных обращениях. Кэш автоматически очищается при изменении коллекции-источника или её элемнтов, если они являются Observable объектами.
