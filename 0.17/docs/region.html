Region

Определение

Обычно регионы определяются в HTML разметке с помощью css-класса x-regon. Элементы регионов должны быть внутри элемента Area (если явно определенных областей нет, то под областью подразумевает элемент body). Application при старте (Application.run) для каждой Area находит все элементы с классом x-regon и создает связанные с ними объекты Region. Объекты Region добавляются в RegionManager, экземпляр которого содержит каждая Area (поле Area.regionManager). Для объекта Region помимо DOM-элемента должно быть задано имя. Имя региона берется из data-атрибута region, либо при его отсутствии, из атрибута id. Примеры определения регионов:

<div class="content x-area" data-area-options="isDefault:true" style="padding-top: 0px;">
    <div class="x-region" id="mainmenu"></div>
    <div class="x-region" data-region="main" data-region-options="behavior:'default-target',navigable:true">
</div>

Помимо имени для региона в разметке могут быть заданы опции - в атрибуте region-options. Содержимое атрибута - JSON-подобное определение пар параметр-значение. Поддерживаются следующие опции:

  • navigable {Boolean} - признак (true) навигируемого региона, в регионе с таким признаком будут открываться все парты области во время клиентской навигации (см. Navigation)
  • behavior {String} - наименование бихейвера региона, который может обрабатывать различные этапы жизнедеятельности региона (открытие/закрытие парта). В качестве наименование бихейвера можно указать поле в глобальном объекте core.regionBehaviors, в котором содержится экземпляр бихейвера. Бихейвер должен реализовывать следующий API:
/**
* @param app {Applicatin}
* @param region {core.composition.Region}
* @param regionDomEl {DOMElement}
*/
attach: function(app, region, regionDomEl) {}

Lifecycle

Регион в один момент времени отображает один парт (по крайней мере сейчас есть только такая реализация). Активация парта происходит с помощью метода:

region.activePart(part, options)

Если в регионе уже есть текущий парт (и он отличается от переданного), то текущий парт выгружается. Для этого у него вызываются два метода: queryUnload и unload. Метод queryUnload вызывается (если в параметрах метода region.activePart не было задано unloadOptions.skipQueryUnload и метод присутствует у парта) чтобы выяснить готов ли парт выгрузиться. Если метод queryUnload парта вернет непустой результат, то он считается описанием причины запрета выгрузки. Если метод queryUnload вернул пустоту (либо не вызывался), то после него вызывается метод unload (опять же если есть). Результат метода unload трактуется как логический признак keepAlive - хранить ли экземпляр парта в регионе или удалить его (по умолчанию).

Если парт еще не существует в регионе, то регион добавляет его в себя и вызывает его метод render. Также регион подписывается на событие statechange парта. Если регион уже содержит экземпляр парта (но тот не активен), то регион активирует парт (делает видимым его DOMElement и вызывает методы activating/activated парта, если они определены).

При добавлении парта к себе регион вызывает у него метод setNavigationService, передавая в него свой экземпляр NavigationService. NavigationService для региона изначально устанавливает RegionManager. NavigationService используется партом для перехода во вложенные контексты, подробней см. NavigationService.
