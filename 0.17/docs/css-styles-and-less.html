CSS Styles and LESS

При разработке WebClient используется LESS - CSS pre-processor. Однако, дистрибутив WebClient поставляется и с less и с css файлами. Прикладной проект может выбирать что ему использовать: css или less.

В рантайме приложение загружает файлы стилей, заданные в конфигурации main.config.json. Подробней см. Bootloader и Page+Layout. Обычно это минимум три файла:

  • "lib/ui/styles/root.css"
  • "content/generic.css"
  • "app/ui/styles/root.css"

lib/root.css включает общие стили. Компилируется из root.less. Файл generic.css используется для билда production-версии с помощью grunt-croc-requirejs. В него включаются все стили из css-файлов, импортируемых используемыми компонентами WebClient через requirejs-плагин xcss. В девелоперской версии стили плагин xcss включает стили непосредственно в тег style на странице. Файл app/root.css принадлежит прикладному приложению.

Кастомизация стандартных CSS

Кастомизация стилей WebClient может быть выполнена двумя основными способами:

  • переопределением css в app/root.css
  • переопределением less переменных с последующей перекомпиляцией css

Первый способ неоптимальный, т.к. переопределение селекторов выполняется в рантайме.

Переопределени переменных less возможно двумя способами:

  • задание переменных в Grunt-таске less
  • замена файла .variables-override.less

При создании проекта с помощью Yeoman-генератора, в Gruntfile.js добавляется таск компиляции less-файлов (из пакета grunt-contrib-less). Таск создается с двумя таргетами - один (dist) для компиляции less из дистрибутива WebClient, другой (app) для компиляции только less из текущего приложения (используется, если не требуется модификация страндарных стилей). Таск примерно следующего содержания:

less: {
            app: {
                src: ['src/app/**/*.less'],
                expand: true,
                rename: function(dest, src) { return dest + src.replace('.less', '.css'); },
                dest: ''
            },
            dist: {
                src: [
                    'dist/dev/lib/** /*.less',
                    'dist/dev/modules/** /*.less',
                    'dist/dev/vendor/**.less'
                ],
                expand: true,
                rename: function(dest, src) { return dest + src.replace('.less', '.css'); },
                dest: ''
            }
        },

В опциях таргета таска less можно определить объект modifyVars, в котором переопределить значения less-переменных. Если требуется модифицировать какие-то переменные LESS из WebClient или Bootstrap, то необходимо запускать оба таргета. Поэтому следует в оба таргета добавить объект options с опций modifyVars (использует фичу less http://lesscss.org/usage/#command-line-usage-modify-variable):

less: {
            app: {
                options: {
                    modifyVars: {
                        "color-main": "#428bca"
                    }
                },
            },
            dist: {
                options: {
                    modifyVars: {
                        "color-main": "#428bca"
                    }
                },
            },
        }

Запуск через алиас "build":

grunt.registerTask('build', function () {
        var tasks = [
            'less:app',
            'clean:dist', 
            'copy:dist',            
            'less:dist'
        ];
        grunt.task.run(tasks);
    });

Второй способ заключается в определении переменных в файле .variables-override.less. Этот файл - заглушка (пустой), он импортируется в самом конце .variables.less, который импортируется все все остальные Less WebClient. Однако, надо учесть, что модификации в этом файле пропадут при следующем обновлении. Поэтому кастомизируемый файл следует хранить в app и копировать таском Grunt при каждом билде. Например:

copy: {
        dist: {
            files: [{ 
                // копируем дистрибутив в dist/dev
                src: '**/*.*',
                expand: true,
                dot: true,
                cwd: 'vendor_components/croc.webClient/src',
                dest: 'dist/dev'
            }, {
                // копируем приложение в dist/dev
                src: '**/*.*',
                expand: true,
                dot: true,
                cwd: 'src',
                dest: 'dist/dev'
            }, {
                // копируем переопределение less в dist/dev
                src: '.theme-override.less',
                dot: true,
                dest: 'dist/dev/lib/ui/styles/.variables-override.less'
            }]
        },
    },

В файле .theme-override.less можно переопределить переменные less, либо импортировать готовую тему:

@import "themes/.theme-blue.less";

В дистрибуте WebClient поставляются файлы тем в папке lib/ui/styles/themes.
