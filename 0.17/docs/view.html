View

View - это парт, который для рендеринга использует некоторый шаблон. В качестве движка шаблонов используется Handlebars.

{toc}

Опции

Наименование Тип Описание
template String|Function Handlebars шаблон. Может быть уже скомпилирован (Function) или представлен в виде текста (String). В последнем случае шаблон сначала компилируется в функцию. Шаблон также можно задать с помощью метода setTemplate (см. далее раздел #Компиляция и production)
viewModel Object ViewModel, которая используется в качестве контекста функции-шаблона. Модель также можно задать с помощью метода setViewModel. Если viewModel не задана до вызова метода render, то в качестве viewModel выступает сам View
unbound Boolean По умолчанию шаблон оборачивается в observable expression, что позволяет автоматически отслеживать изменения во viewModel. При изменении viewModel вызывается повторный рендеринг (rerender), функция-шаблон при этом вычисляется заново.

Компиляция и production

При формировании production версии (с помощью таска rjs из grunt-croc-requirejs) Handlebars-шаблоны, импортируемые с помощью RequireJS-плагина xhtmpl компилируются в js-код. Кроме того, сам плагин xhtmpl оптимизируется т.о. чтобы не использовать полную версию Handlebars. По умолчанию, плагин xhtmpl - это единственный компонент который использует полную версию Handlebars. Остальные компонент (фактически View) зависят от runtime-версии - облегченной версии без компилятора. Переключение версии Handlebars происходит при билде с помощью rjs из grunt-croc-requirejs - если для таска задан параметр:

requireConfigOutput: {
        paths: {
            handlebars: "vendor/handlebars/handlebars.runtime"
        }
    }

Т.о. если для production-версии в require.config.json для модуля "handlebars" задано отображение на "handlebars.runtime", то динамическая компиляция шаблона (заданного через View.setTemplate) работать не будет. Рекомендуется хранить шаблоны в отдельных hbs-файлах и импортировать их с помощью плагина xhtmpl. Если же в приложении требуется использовать динамическую генерацию шаблона, то необходимо использовать полную версию Handlebars. В версии 0.15 стандартная компиляция Handlebars дополнена фичей удаления лишних пробелов и переносов строк. Если между окончанием тега (>) или хелпера (}}) и начало другого тега (<) или хелпера ({ {) содержатся только переносы строки и пробельные символы, то они удаляются (с помощью регулярного выражения). Это позволяет писать форматированные шаблоны с отступами и переносами строк, не боясь за качество выходного HTML. Однако, иногда бывает нужно сохранить разметку без изменений. Для этого введен хелпер pre: все, что внутри него, останется без изменении. Пример:

<div>

    <span>Переносы строк будут вырезаны<span>

</div>
{{#pre}}<pre>

    <span>Переносы строк останутся без изменений<span>

</pre>{{/pre}}

Модификация Handlebars

В стандартной поставке Handlebars умеет использовать только поля JavaScript-объектов или их свойства, объявленные через defineProperty (или подобными способами). Однако, в WebClient свойства объектов (доменных и не только) реализованы через методы. Чтобы задействовать такие свойства, Handlebars был кастомизирован. А именно изменен метод Handlebars.JavaScriptCompiler.prototype.nameLookup: в нашей версии он сводится к вызову core.lang.Observable.get. Таким образом, если в шаблоне используется метод, то он будет автоматически вызван (без аргументов), как если бы это было простое поле. Синтаксис в шаблоне один и тот же как для полей, так и для методов (без дополнительных скобок). Однако, иногда не нужно сразу вызывать метод в шаблоне. Например, это может потребоваться, чтобы передать функцию внутрь хелпера как опцию. Рассмотрим на примере хелпера format. Он принимается опцию formatter - функцию, возвращающую строковое представление объекта. Если в шаблоне вызывать этот хелпер следующим образом:

{{format value formatter=options.formatter}}

то функция options.fomatter будет сразу же вызвана и в хелпер попадет уже ее результат - скорее всего все упадет. Чтобы этого избежать предлагается использовать следующий синтаксис:

{{format value formatter=options.['formatter']}}

Вместо одинарных можно использовать двойные кавычки.

Helpers

Вместе со View поставляется набор Handlebars-хелперов, облегчающих написание шаблонов:

data-bind

Выполняет data binding свойства viewModel и DOM-элемента. arguments: нет block: нет hash: | target | Целевой элемент (подробнее) | Не обязательно. Добавлен в версии 0.10. | | Прочие опции | Остальные опции рассматриваются как словарь устанавливаемых binding-ов. Ключ - наименование html-байдинга (например, text, value, valueLive и т.д.). Значение - observable expression (см. xfwajax:/2012/05/10/Универсальный data-binding). | Должна быть хотя бы одна такая опция | Примеры: 

<input type="text" {{data-bind value="lastName"}} />

<input type="text" {{data-bind value="this.organizer().firstName() + ' ' + this.organizer().lastName()" hidden="!this.organizer()"}} />

command-bind

Выполняет command binding команды и DOM-элемента. argument: экземпляр команды block: нет hash: | target | Целевой элемент (подробнее) | Не обязательно. Добавлен в версии 0.10. | | canBeEmpty | true/false | по умолчанию, если аргумент не возвращает объект команды, возникает исключение, чтобы это не происходило можно задать опцию canBeEmpty=true - в этом случае для пустого поля байндинг будет игнорироваться | | Прочие опции | Параметры, передавемые к команду | Не обязательно | Пример: 

<button {{command-bind cmdSave item=currentItem}}>Save</button>

render

Отображает уже существующий парт. argument: экземпляр парта block: нет hash: | target | Целевой элемент (подробнее) | Не обязательно. Добавлен в версии 0.10. | | tag | Тег обрамляющего элемента | Не обязательно, по умолчанию span | | cssClass | Css-класс обрамляющего элемента | Не обязательно | Пример:

{{render body}}

observe

Создает независимую область отслеживания изменений. Если изменения во viewModel коснулись только части DOM внутри хелпера observe, то будет перерисована только часть, а не весь View.

На самом деле просто создается дочерний View argument: дочерняя viewModel block: шаблон, изменения в котором будут отслеживаться отдельно hash: | target | Целевой элемент (подробнее) | Не обязательно. Добавлен в версии 0.10. | | tag | Тег обрамляющего элемента | Не обязательно, по умолчанию span | | cssClass | Css-класс обрамляющего элемента | Не обязательно | Пример: 

<div style="height:200px">
{{#observe this}}
    <span>{{description}}<span>
{{/observe}
<div>

unbound

Создает область без отслеживания изменений. Таким образом, если изменения коснулись только части DOM внутри хелпера unbound, то перерисовка View не будет выполняться.

На самом деле просто создается дочерний View с опцией unbound: true. В остальном аналогичен хелперу observe.

each

Переопределяет стандартный Handlebars-хелпер each, добавляя поддержку ObservableCollection.

observe-each

Вызывает хелпер observe для каждого элемента коллекции. В отличие от each при изменении одного элемента коллекции будет перерисован только этот элемент, а не весь View. argument: дочерняя viewModel block: шаблон для элемента коллекции hash: | target | Целевой элемент (подробнее) | Не обязательно. Добавлен в версии 0.10. | | tag | Тег обрамляющего элемента | Не обязательно, по умолчанию span | | cssClass | Css-класс обрамляющего элемента | Не обязательно | Пример: 

<div>
{{#observe-each comments}}
    <p>{{text}}<p>
{{/observe}
<div>

expr

Подставляет результат выражения. argument: текст javascript выражения (указывается) block: нет hash: нет Пример: 

<span>{{expr "this.getFormatted('created')"}}</span>

if-not-last

Подставляет блок, если текущий элемент не последний в коллекции. Должен применяться внутри хелперов, итерирующих коллекции (например, each). arguments: нет block: подставляемый шаблон hash: | total | Число элементов в коллекции | Обязательно | Пример: 

{{#if-not-last total=../pages.count}}
<span class="wizard-arrow"></span>
{{/if-not-last}}

compare

Подставляет блок, если операция сравнения возвращает true. arg1: левый операнд для сравнения arg2: строка с оператором сравнения. Может иметь одно из следующих значений: ==, ===, !=, !==, <, >, <=, >= или typeof arg3: правый операнд для сравнения

Может быть задано только два аргумента. В этом случае они сравниваются оператором ===. block: подставляемый шаблон hash: нет Пример: 

{{#compare pages.count ">" 1}}
    <div class='x-editor-root x-editor-root-tabbed'></div>
{{else}}
    <div class='x-editor-root x-editor-root-single'></div>
{{/compare}}

push-id

Хэлпер создает уникальный id элемента, сохраняет и выводит его. Подробнее Пример: 

<label for="{{push-id}}">
    <input type='checkbox' id="{{pop-id}}" />
</label>

peek-id

Хэлпер выводит уникальный id элемента, созданный ранее с помощью push-id. Подробнее Пример: 

<label for="{{push-id}}">
    <input type='checkbox' id="{{peek-id}}" {{data-bind checked="someProp" target=":pop-id"}} />
</label>

pop-id

Хэлпер выводит уникальный id элемента, созданный ранее с помощью push-id (аналогично peek-id), но после этого очищает сохраненный id. Подробнее Пример: 

<label for="{push-id}">
    <input type='checkbox' id="{pop-id}" />
</label>

toggleClass

Создает binding между наличием у элемента css класса (binding cssClassToggle) и значением observable expression. argument: нет block: нет hash: задает пары имя-значение, где имя - это имя css класса, наличие которого у dom-элемента ставится в соответствие от истинности observable-выражения, задаваемого значением. Пример: 

<span {{toggleClass -disabled="disabled"}}></span>

format

Выводит форматированное представление объекта argument: форматируемый объект (если не задан, то используется текущий контекст) block: нет hash: | formatter | Функция-форматтер или наименование форматтера для доменного объекта. | Необязательно | Если задана функция, то она вызывается без аргументов в this-контексте форматируемого объекта (аргумент хелпера) | Пример: 

{{format value formatter=options.['formatter']}}

Важно использовать именно такой синтаксис с квадратными скобками и кавычками. Подробнее выше.

or

Возвращает логическое OR для переданных аргументов. Удобно использовать как subexpression. argument: два и более аргумента для вычисления логического OR block: нет hash: нет Пример: 

{{#if (or firstProp secondProp thirdProp.nestedProp)}}
    {{!-- Some block --}}
{{/if}}

and

Возвращает логическое AND для переданных аргументов. Удобно использовать как subexpression. argument: два и более аргумента для вычисления логического AND block: нет hash: нет Пример: 

{{#if (and firstProp secondProp thirdProp.nestedProp)}}
    {{!-- Some block --}}
{{/if}}

External helpers

Вышеперечисленные хелперы являются наиболее общими и регистрируются непосредственно в модуле View. Другие модули также регистрируют некоторые полезные хелперы:

Core helpers

part

Создает парт и отображает его. arguments: нет block: нет hash: | target | Целевой элемент (подробнее) | Не обязательно. Добавлен в версии 0.10. | | tag | Тег обрамляющего элемента | Не обязательно, по умолчанию span | | cssClass | Css-класс обрамляющего элемента | Не обязательно | | class | Класс парта в неймспейсе core | Должна быть задана одна из опций class или name | | name | Имя зарегистрированного парта | Должна быть задана одна из опций class или name | | viewModel | ViewModel для установки созданному парту | Не обязательно, по умолчанию используется текущий контекст | | Прочие опции | Опции, передаваемые в конструктор парта | Не обязательно | Пример: 

{{part class="ui.MenuButtonsPresenter" viewModel=menuList}}

xconfig

Подставляет значение конфигурационного параметра из объекта core.Application.current.config. argument: имя параметра block: нет hash: нет Пример: 

<a href="#">{{xconfig "appName"}}</a>

res

Подставляет значение локализованной строки из ресурсов (см. i18n). argument: ключ в словаре ресурсов block: нет hash: нет Пример: 

<label class="radio">{{res "loginDialog.windows_account"}}</label>

if-feature

Подставляет блок, если заданная фича поддерживается текущим окружением. arguments: наименование фичи block: подставляемые шаблон hash: нет Пример: 

{{#if-feature "cssanimations"}}
    <div class="x-icon-sync x-icon-anim-rotating x-icon-48"></div>
{{else}}
    <div class="x-img-waiting-48"></div>
{{/if-feature}}

waiting-icon

Вывод анимированную иконку, символизирующую ожидание. Используется для кросс-браузерной вставки анимации, т.к. IE8 не поддерживает css-анимацию, с помощью которой реализована анимированная иконка по умолчанию. waiting-icon hash: | size | размер иконки | Не обязательно. По умолчанию 48px |

propImageUrl

Возвращает url для тега IMG с картинкой из бинарного свойства объекта. Контекст хелпера (this) должен быть доменным объектом. arguments: имя свойства block: нет hash: | width | Ширина картинки | Не обязательно. | | height | Высота картинки | Не обязательно. | Пример: 

<img src="{{propImageUrl "photo" width="128" height="128"}}" width="128" height="128" style="max-width: 128px; max-height: 128px;"></img>

Результат:

<img src="/droneshop/api/_file/binaryPropValue?entityType=CatalogItem&amp;objectId=6b11ab13-76de-4411-aefc-589c124e35b2&amp;propName=photo&amp;width=128&amp;height=128" width="128" height="128" style="max-width: 128px; max-height: 128px;">

propImage

Возвращает тег IMG с url'ом для загрузки картинки из бинарного свойства объекта. Контекст хелпера (this) должен быть доменным объектом. arguments: имя свойства block: нет hash: | lazy | true для поддержки lazy load | Не обязательно. По умолчанию false. | | width | Ширина картинки | Не обязательно. | | height | Высота картинки | Не обязательно. | Пример: 

{{propImage "photo" width="128" height="128"}}

Результат аналогичен примеру для propImageUrl. Пример: lazy load

{{propImage "photo" lazy="true" width="128" height="128"}}

Результат:

<img src="data:image/png;base64,..." data-original="/droneshop/api/_file/binaryPropValue?entityType=CatalogItem&amp;objectId=6b11ab13-76de-4411-aefc-589c124e35b2&amp;propName=photo&amp;width=128&amp;height=128" width="128" height="128" style="max-width: 128px; max-height: 128px;">

Как только картинка попадет во вьюпорт окна, т.е. станет видимой, url из атрибута data-original будет скопирован в src. Для lazy-load используется плагин jquery.lazyload

IconProvider helpers

icon

Отображает иконку. arguments: нет block: нет hash: | name | Наименование иконки, зарегистрированной в xfwajax:/2012/12/28/IconProvider | Обязательно | Пример: 

<a href="#">{{icon name="remove"}}</a>

EditorPagePresenter helpers

pe

Отображает редактор свойства. Если нужного редактора свойства нет на странице, то он создается. arguments: нет block: нет hash: | target | Целевой элемент (подробнее) | Не обязательно. Добавлен в версии 0.10. | | tag | Тег обрамляющего элемента | Не обязательно, по умолчанию span | | cssClass | Css-класс обрамляющего элемента | Не обязательно | | name | Имя свойства из viewModel, для которого отображется PE | Обязательно | | viewModel | Дочерняя viewModel | Не обязательно, по умолчанию используется текущий контекст | | keyPrefix | Префикс, который добавляется к ключу, под которым объект редактора свойства помещается в словарь EditorPage.editors | Не обязательно, по умолчанию используется имя свойства | | Прочие опции | Метаданные создаваемого PE | Не обязательно | Пример: 

<div class="span4"><label>Текст вопроса:</label></div>
<div class="span8">{{pe name="text"}}</div>

>peContainer

Отображает разметку для уже существующего редактора свойства. Разметка включает сам редактор свойства, label для него (с css-классом x-pe-label) и обрамляющий их контейнерный элемент (css-класс x-pe-container). arguments: Экземпляр редактора свойства block: нет hash: нет

Это не хелпер, а patrial шаблон. Пример: 

{{> peContainer editors.title}}
{{> peContainer editors.organizer}}

peLayout

Для всех вложенных partial шаблонов #>peContainer задает:

  • отношение ширины лейбла к ширине редактора свойства
  • флаг "не выводить метку" arguments: нет block: шаблон, содержащий один или несколько partial-ов #>peContainer hash: | columnRatio | Отношение ширины лейбла к ширине редактора свойства в виде "N:M" | | noLabel | Флаг "не выводить метку" |

Числитель N и знаменатель M в columnsRatio - это фактически суффиксы для классов колонок в bootstrap-овском гриде. Поэтому в сумме они должны давать 12 (или меньше, тогда строка будет не на всю ширину). Пример: 

{{#peLayout columnsRatio="2:10"}}
    {{> peContainer editors.title}}
    {{> peContainer editors.organizer}}
{{/peLayout}}

#>peContainer берет ширину лейбла и редактора свойства из обрамляющего хелпера peLayout, если он есть, или из полей labelColumnRatio и peColumnRatio страницы.

peContainer

Является короткой формой записи для #peLayout и вложенного #>peContainer. arguments: Экземпляр редактора свойства block: нет hash: | columnRatio | Отношение ширины лейбла к ширине редактора свойства в виде "N:M" | | noLabel | Флаг "не выводить метку" | Пример: 

{{peContainer editors.title columnsRatio="2:10" noLabel=true}}

То же самое:

{{#peLayout columnsRatio="2:10" noLabel=true}}
    {{> peContainer editors.title}}
{{/peLayout}}
