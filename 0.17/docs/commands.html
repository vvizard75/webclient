Commands

Команда - это полный аналог WPF команд. Это объект со следующим API:

execute: function (args) {}
canExecute: function() {}

Фактически это объект обертка вокруг колбэка, который может хранить состояние доступности (canExecute). Функция canExecute реализует паттерн свойства:

// set:
canExecute(true);
// get (вызов без параметров):
return canExecute();

Перед вызовом команды (execute) необходимо проверять свойство canExecute. Свойство canExecute просто хранит признак доступности и генерирует событие ("change:canExecute") при изменения значения - используется стандартный патерн Observable-объекта (объект команды наследует от core.lang.Observable).

Также для команды могут быть заданы поля:

  • name - опциональная идентификация команды, используется для автоматической привязки пунктов меню к командам.
  • title - опциональный заголовок команды, используется в качестве наименования пункта меню при автоматической привязки пунктов меню к командам.
  • params - объект параметров команды, данный объект объединяется с параметрами, передаваемыми в execute

По завершению метода execute для объекта команды генерируется событие "executed". Параметры события - параметры метода execute.

Использование команды

Сценарии использования команд:

  • явное, как обертку вокруг колбэка
  • связывание с пунктами меню, пункт меню всегда связан с командой, с одной командой может быть связано несколько пунктов меню
  • связывание с html-элементами с помощью байндинга: подписка на событие click элемента, при котором вызывается execute, создание байндинга между свойство canExecute команды и свойством enable html-элемента. Для создания командного байндинга следует использовать метод binding.commandBind (см. command-binding).

Создание команды

Для создания команды следует использовать метод createCommand модуля core или core.commands:

var cmd = core.createCommand({});

В качестве параметров методу надо передать объект-спецификацию команды - объект может содержать (все поля опциональны):

  • поле canExecute - начальное значение свойство canExecute
  • функцию canExecute - данная функция будет использоваться как свойство canExecute объекта команды (убедитесь, что вы реализовали функцию в соответствии с патерном "свойство")
  • метод execute - метод будет вызываться из метода execute объекта команды в контексте объекта команды (т.е. this - всегда будет командой вне зависимости от способа вызова метода)

Привязка команд и меню

Привязка пунктов к командам может быть выполнена изначально при создании - создание меню по метаданным, где для пунктов уже заданы команды:

var cmd = core.createCommand({});
var menu = new core.ui.Menu({items: [ { name: "Edit", title: "Редактировать", command: cmd} ]});

Также привязка может быть выполнена позже: мы отдельно создали меню по метаданным и имеем коллекцию команд:

var commands = {
  Edit: core.createCommand({ execute: function(args) {} }),
  Create: core.createCommand({ execute: function(args) {} })
};
var menu = new core.ui.Menu(
  {items: [
    { name: "Edit", title: "Редактировать" },
    { name: "Create", title: "Создать" }
  ]
});
menu.bindToCommands(commands);

В методе Menu.bindToCommands выполняется привязка пунктов меню, для которых еще не задана команда, к командам по совпадению наименований (name)

Параметры команды

Для команды могут быть заданы параметры. Они передаются в метод execute, объединяясь с явно заданными параметрами. Параметры команды задаются в поле params. Пример:

var cmdEdit = core.createCommand({
  execute: function(args) {
    /* args будет представлять из себя объект:
     {
       param1: "value1",
       param2: "value2_runtime",
       param3: "value3_runtime",
     }
    */
  }
});
cmdEdit.params = { param1: "value1", param2: "value2" };

cmdEdit.execute({param2: "value2_runtime", param3: "value3_runtime"});

Помимо задания параметров команды в самой команде, их также можно задать:

  • в пункте меню, который связывается с командой - в этом случае параметры копируются (объединяются) из пункта меню в команду (при создании меню или bindToCommands)
  • с помощью байндинга (см. command-binding)

command-binding

Байндинг между html и командой может быть создан следующими способами:

  • из шаблона с помощью хелпера command-bind:
<button class="btn" {{command-bind viewmodel.cmdAddQuestion}}>Добавить вопрос</button>

Аргументом хелпера является свойство, возвращающее объект команды. Все остальные аргументы (в виде параметр="значение") трактуются как параметры команды, например:

<button style="margin-right: 20px" class="btn" {{command-bind ../viewmodel.cmdRemoveQuestion object=this}}>Удалить</button>
  • явно, используя API модуля binding - метод commandBind:
binding.commandBind = function (element, cmd, args) {}

Первый способ сводится ко второму.

Байндинг между html и командой означает следующее:

  • свойство enabled html-элемента привязывается к canExecute команды (т.е. при установке canExecute(false) элемент блокируется)
  • событие "click" html-элемента привязывается к методу execute (т.е. при клике на элемент выполняется метод execute команды)

    Если кому-то нужны другие события, обращайтесь.
