Archived-Build 0.10

Описание формирования production-версии приложения для WebClient 0.10 и ниже. Неактуально начиная с версии 0.11

Формирование production билда приложения

В процессе билда клиентской части приложения на xfw3.webClient выполняются следующие основные действия:

  1. Объединение client\*.js файлов приложения в несколько файлов с разбиением по слоям и последующая их минификация.

Задействуются только файлы, указанные как RequireJS-зависимости для основных модулей. По умолчанию основными модулями считаются все .js файлы, находящиеся непосредственно в директории client (без поддиректорий). Прочие .js файлы остаются нетронутыми.

  1. Объединение content\*.css файлов приложения в content\generic.css последующая минификация css.

Минифицируются как объединенный файл content\generic.css, так и исходные content\*.css файлы (могут подключаться напрямую через тег .

В качестве инструмента используется оптимизатор из состава RequireJS (версия для nodejs).

Для выполнения билда необходимо под nodejs выполнить javascript-файл make.js находящийся в директории tools\build папки пакета (packages\XFW3.WebClient.X.Y.Z\tools\build\make.js). Параметрами скрипта являются пути к директории приложение (с поддиректориями client и content) и output директория, в которой формируется оптимизированное клиентское приложение. Дополнительно можно указать временную директорию, использующуюся во время билда, и перечень основных модулей. Использование:

node.exe make.js -input=<input dir> -output=<output dir> [-temp=<temp dir>] [-module=<module name>]*");

Пример:

node.exe make.js -input=../../Repo/Src/Server -output=../../Repo/Src/Server.Optimized -module=main -module=report-main");

Ранее бОльшая часть работы выполнялась на powershell скриптом make.ps1. Это файл оставлен для обеспечения обратной совместимости с прочими скриптами, но теперь это просто "пускалка" для make.js.

Результат выполнения билда в output директории:

  • директория сontent;
    • объединенный файл generic.css
    • оптимизированные css-файлы, не указанные как зависимости модулей
    • прочие файлы
  • директория client:
    • main.config.js
    • vendor-layer.js
    • lib-layer.js
    • vendor/require.js
    • модифицированные основные модули (main.js и report-main.js)
    • js-файлы, не указанные как зависимости модулей
    • прочие файлы

Настройки билда

Скрипт make.js поддерживает задание опций командной строки в виде -option=value. Также можно задавать опции в виде JSON в конфигурационном файле. Имя файла по умолчанию - make.config.json, может быть изменено через командную строку опцией -config. Если одна и та же опция задана и в конфигурционном файле, и в командой строке, то значение из командной строки имеет приоритет. Полный список опций можно увидеть, запустив скрипт без параметров. {expand:Все опции}

USAGE: node.exe make.js [-<option>=<value>]*
Options:
        -config: Build config file [make.config.json]
        -input: Input directory
        -output: Output directory
        -temp: Temporary directory [~temp]
        -modules: Add main module [<All .js files directly under clientBase directory>]
        -mainConfig: Application main config file (relative to <input>) [client/main.config.json]
        -clientBase: Subdirectory with *.js files (relative to <input>) [client]
        -contentBase: Subdirectory with *.css files (relative to <input>) [content]
        -optimizeJs: Optimization method of *.js files [uglify]
        -optimizeCss: Optimization method of *.css files [standard.keepLines]
Configuration file:
{
        "input": "<specify value>",
        "output": "<specify value>",
        "temp": "~temp",
        "modules": [ "name1", "name2", ... ],
        "mainConfig": "client/main.config.json",
        "clientBase": "client",
        "contentBase": "content",
        "optimizeJs": "uglify",
        "optimizeCss": "standard.keepLines"
}
Example:
        node.exe make.js -input=../../Repo/Src/Server -output=../../Repo/Src/Server.Optimized -modules=main,report-main

{expand}

Пути к модулям, используемые RequireJS, а также зависимости между файлами, не являющимися AMD-модулями (shim), прописываются в файле client\main.config.json в секции require.

Формирование Web Deploy пакета.

Для создания пакета развертывания необходимо запустить билд проекта с соответствующим target=Package. Например:

msbuild Arms.WebServer.csproj /t:Package /p:Configuration=Release /p:PackageSources="http://nuget.rnd.croc.ru/nuget"

Чтобы при формировании WebDeploy-пакета была выполнена оптимизация клиентских скриптов, необходимо выполнение двух условий:

  • в csproj-файле должна быть переменная ClientMakeScriptPath, указывающая на файл make.ps1
  • рядом с csproj-файлом проекта должен находится файл ИмяПроекта.wpp.target (т.е. если Server.csproj, то Server.wpp.targets)

Оба этих пункта автоматически будут выполнены, при установке NuGet-пакета XFW3.WebClient (создается переменная и wpp.targets файл).

В процессе выполнения target'а Package, после копирования файлов во временную директорию и перед их архивацией будет выполнен target с именем "Compress client files before packaging". Данный target произведет действия, описанные выше в "Формирование production билда приложения".

Стоит заметить, что оптимизация клиентского кода всегда выполняется при условии Release конфигурации или при наличии true параметра OptimizeClient. Таким образом команда

msbuild Arms.WebServer.csproj /t:Package /p:Configuration=Debug /p:OptimizeClient=true /p:PackageSources="http://nuget.rnd.croc.ru/nuget" /p:VisualStudioVersion=11.0

аналогична приведенной выше.

Файл ИмяПроекта.wpp.targets также содержит импорт targets-файла WebDeploy.Custom.wpp.targets, который предназначен для кастомизации формирования WebDeploy-пакета без необходимости изменения ИмяПроекта.wpp.targets. Изначально он содержит указания игнорирования папки Files и файла app_offline.htm при синхронизации.

WebDeploy.Custom.wpp.targets содержит вызов скрипта make.ps1 из nuget-пакета. Путь до скрипта прописывается в свойство ClientMakeScriptPath, которое добавляется в файл проекта (csproj) при установке пакета XFW3.WebClient. Скрипт make.ps1 фактически запускает make.js через node. Target формирования клиентского дистрибутива из wpp.targets-файла поддерживает следующие параметры:

Параметр Значение Значение по умолчанию Эквивалент в main.js
OptimizeClient True/False False нет
ClientModulesToBuild список имен main-модулей через "," пусто modules
ClientBuildConfig имя файла с настройками билда make.config.json config
NodePath путь до node.exe пусто нет

Troubleshooting

При установке WebDeploy-пакета может возникнуть ошибка:

The application pool that you are trying to use has the 'managedRuntimeVersion' property set to 'v4.0'. This application requires 'v4.5'.

Возникает она, если пакет был сформирован из проекта с TargetFrameworkVersion=4.5. Избавиться от нее можно:

  • при формировании пакета (build /t:Package) задать параметр /p:IgnoreDeployManagedRuntimeVersion=True - он отключит проверку на совместимость версий CLR
  • при формировании пакета задать параметр /p:VisualStudioVersion=11.0
