SlickObjectListPresenter - inline editing

inline-editing area: Docs section: parent: slickobjectlistpresenter order: 0 tags:

root: false

В версии 0.13 добавлена новая фича - inline (in-place) редактирование списка (а-ля Excel). Чтобы задействовать эту возможность, в приложении нужно импортировать модуль lib/ui/slick/SlickInlineEditAddon. Это addon, который расширяет SlickObjectListPresenter, добавляя новый функционал. Кроме того, в метаданных списка нужно задать опцию inlineEdit. Эта опция также делает список редактируемым.

Если модуль lib/ui/slick/SlickInlineEditAddon не импортирован, то задавать опцию inlineEdit бессмысленно.

В списке с опцией inlineEdit клик по активной ячейке или нажатие Enter/F2 приведут к открытию редактора свойства (PE) поверх активной ячейки. Метаданными для PE выступают метаданные доменного свойства, которое соответствует активному столбцу. Если столбец не содержит данных свойства (например, checkbox), то PE для такого столбца не будет открываться. Кастомизировать выводимый PE можно, задав опцию editor для колонки. Значение опции - это дополнительные метаданные для PE. Например, укажем количество строк для текстового свойства:

var list = core.ui.ObjectList.create(app, {
                // other options
                columns: [
                    // other columns
                    {
                        name: "description",
                        editor: { rows: 3 }
                    }
                ],
                inlineEdit: true
            });

Так же можно отключить inline редактирование заданного столбца. Для этого нужно в качестве значения опции editor задать любое falsy значение, кроме undefined (например, false или null):

var list = core.ui.ObjectList.create(app, {
                // other options
                columns: [
                    // other columns
                    {
                        name: "description",
                        editor: null
                    }
                ],
                inlineEdit: true
            });

Можно явно задать класс реализации PE через опцию PropertyEditor (внутри editor):

var list = core.ui.ObjectList.create(app, {
                // other options
                columns: [
                    // other columns
                    {
                        name: "organizer",
                        editor: { PropertyEditor: core.ui.peObject } 
                    }
                ],
                inlineEdit: true
            });

Также в метаданные PE добавляется опция contextName: "inline". Ее можно использовать при выборе реализации PE по умолчанию в PropertyEditor.DefaultMapping.

Кастомизировать внешний вид inline PE можно также с помощью CSS. После рендеринга PE к его корневым DOM-элементам добавляется CSS класс .x-pe-inline. Под "корневыми" элементами здесь понимаются DOM-элементы, являющиеся непосредственными детьми элемента-контрейнера. В качестве контейнера выступает специальный <div> с css-классом .x-pe-inline-container, который вставляется внутрь ячейки грида при открытии PE. Таким образом, можно определять специфический вид PE, если он открыт как inline редактор ячейки списка. Например:

.x-pe-inline.x-pe-bool-switch {
    // some style properties
}

Класс .x-pe-inline добавляется уже после рендеринга PE. Однако, иногда уже в процессе рендеринга нужен полностью сформированный стиль элементов. Например, peObjectList (с презентером peSlickObjectListPresenter) при рендеринге вычисляет ширину грида по примененному стилю. В этом случае можно использовать селектор типа .x-pe-inline-container > .x-pe-object-list. Дело в том, что при отображении PE внутри ячейки грида создается дополнительный контейнерный элемент с css-классом .x-pe-inline-container.
