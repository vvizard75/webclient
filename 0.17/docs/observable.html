Observable

Класс Observable

Класс Observable, как не трудно догадаться из его названия, реализует паттерн наблюдатель). Использование данного паттерна, позволяет упростить зависимости между объектами, и выделить некий контракт уведомления одних объектах об изменениях в других.

Как работает Observable?

Observable представляет собой объект, с некоторым количеством свойств, уведомляющий подписчиков об изменении этих свойств. Так как настоящие свойства в JavaScript в общем случае не поддерживаются, то вместо них используются специальные функции, которые будучи вызываны без аргументов возвращают текущее значение свойства, с одним аргументом - устанавливают новое значение. Подробнее: Реализация свойств.

Как работать с Observable?

Рассмотрим подробнее, какие возможности нам предоставляет класс Observable. Ключевыми методами, для работы с Observable являются:

Названиие Сигнатура Описание
bind function (eventName, callback, context) Подписывает обработчик callback на событие eventName. Если задан параметр context, то обработчик будет вызван в контексте этого объекта (т.е. context будет выступать в качестве this). Если context не задан, то обрабочик будет вызываться в контексте текущего observable-объекта.
unbind function (eventName, callback, context) Отписывает обработчик callback от события eventName. Обработчик отписывается, только если он был подписан с тем же значением параметра context.

Если задано только eventName, то от события eventName отписываются все подписанные обработчики.

Если не заданы eventName, callback и context, то все подписанные обработчики отписываются от всех событий.

Если заданы eventName и context, то от события eventName отписываются все обработчики контекста.

Если задан только context, то отписываются все обработчики контекста от всех событий. | | subscribe | function (eventName, callback, context) | Подписывает обработчик callback на событие eventName (аналогично методу bind) и возвращает disposable объект, который отписывает этот обработчик в методе dispose. | | trigger | function (eventName) | Возбуждает событие eventName, вызывая все подписанные на него обработчики. Все аргументы метода, кроме первого (eventName), передаются в обработчики. | | bound | function (eventName) | Проверяет, что на событие eventName подписан хотя бы один обработчик. Если имя события не задано, то проверяет, что хотя бы один обработчик подписан на любое событие. | | get | function (name) | Получает значение свойства name. Также значения можно получать напрямую, вызывая соответствующие функции (см. Реализация свойств). | | set | function (name,v) | Устанавливает значение свойства name в значение v. Также значения можно устанавливать напрямую, вызывая соответствующие функции (см. Реализация свойств) | | declareProp | function (name, initial) | Создает в Observable-объекте свойство с именем name b начальным значением initial. Вызова событий при создании свойства не происходит. Фактически является вызовом accessor, с сохранением результата в поле name объекта. |

Как уже становится понятно из описания выше, для того, чтобы подписаться на изменения свойств нужно использовать метод bind (либо subsribe), а для того, чтобы перестать получать уведомления - метод unbind.

События генерируемые Observable-объектом имеют следующие имена:

Название Аргументы Описание
set (obj, {prop:name,value:v,oldValue:old}) Вызывается при каждом вызове сеттера любого свойства, вне зависимости от того, совпадает старое значение свойства с новым или нет. В качестве первого аргумента в коллбэк передается сам Observable-объект, в качестве второго объект, описывающий изменение свойства.
get (obj, {prop:name,value:v}) Вызывается при каждом вызове геттера любого свойства В качестве первого аргумента в коллбэк передается сам Observable-объект, в качестве второго объект, описывающий получение свойства.
change (obj, {prop:name,value:v,oldValue:old}) Вызывается при каждом изменении любого свойства. Вызывается только если значение свойства изменилось. Параметры аналогичным событию set.
change:propertyName (obj, newValue, oldValue) Вызывается при каждом изменении свойства propertyName. Вызывается только если значение свойства изменилось. В качестве первого аргумента в коллбэк передается сам Observable-объект, в качестве второго новое значение свойства, а третьего - предыдущее.

Также вам возможно понадобятся статические методы, класса Observable:

Наименование Сигнатура Описание
isObservable function (obj) Проверяет, является ли объект obj Observable объектом
get function (obj, name) Получает значение свойства name объекта obj
accessor function (name, field) Генерирует функцию-свойство, которая будет геттером и сеттером нового свойства (подробнее см. Реализация свойств). name - имя нового свойства (используется в событиях), field - имя поля для хранения значения свойства. Если параметр field не задан, то используется "_" + name.
getter function (name, field) Генерирует функцию-свойство, которая будет геттером нового свойства (подробнее см. Реализация свойств). name - имя нового свойства (используется в событиях), field - имя поля для хранения значения свойства. Если параметр field не задан, то используется "_" + name.

Как сделать свой Observable?

Это очень просто. Для этого вам понадобится:

  1. Создать новый объект класса Observable (не обязательно использовать Observable напрямую, вы можете Наследоваться от Observable)
  2. Создать свойства с помощью accessor, getter или declareProp

Либо создать Observable на основе существующего объекта с помощью метода core.lang.observe(obj). Данный метод принимает на вход JavaScript объект и возвращает новый Observable объект, в котором все поля исходного объекта заменены на одноименные функции-свойства.

Создание свойств:

//с помощью declareProp
obj.declareProp("name","Иван");
obj.name('Сидор');
var myName = obj.name();

//с помощью accessor
obj.lastName = core.lang.Observable.accessor("lastName");
obj.lastName('Иванов');
var myLastName = obj.lastName();
