Distribution

WebClient состоит из двух частей: серверной и клиентской части. Серверная часть - это .NET сборки, распространяющиеся в виде NuGet-пакетов. Сервервая часть WebClient для работы требует XFW3. Клиентская часть WebClient - это набор js/css скриптов и других артефактов. Клиентская часть зависит только от api сервера. Распространяется клиентская часть как в виде NuGet-пакетов, так и как bower-компонент через приватный bower-репозиторий.

NuGet-пакеты

NuGet-пакеты распространяются через приватный репозиторий http://nuget.rnd.croc.ru/nuget. Подробней см. xfw3:Release Management

Пакет Описание Зависимости
XFW3.WebClient.Server Серверная часть WebClient XFW3.Server, XFW3.Core, пакеты ASP.NET MVC и WebApi, Ninject, Newtonsoft.Json
XFW3.WebClient.Client Клиентская часть нет
XFW3.WebClient Helper-пакет для одновременной установки Server и Client XFW3.WebClient.Server, XFW3.WebClient.Client
XFW3.WebClient.Sample Sample-пакет для создания заготовки проекта Server + Client XFW3.WebClient.Client
XFW3.WebClient.Transfer Transfer-модуль - серверная часть для Transfer XFW3.WebClient.Server, XFW3.Transfer.Server
XFW3.WebClient.Reporting Reporting-модуль - серверная часть для поддержки отчетов XFW3.WebClient.Server, XFW3.ReportService.Server

Bower

Клиентская часть распространяется как bower-пакет croc.webClient. Пакет хранится в нашем приватном репозитории по адресу http://10.20.12.101:8001. Bower - это менеджер пакетов для front-end проектов. Bower сам является npm-пакетом (выполняется под node.js). Поэтому перед использованием он должен быть установлен (рекомендуется глобально): 

npm install bower -g

Подробнее о bower: http://bower.io/

Bower для работы требует Git.

По умолчанию bower (также как и npm) использует глобальный репозиторий для установки пакетов и их зависимостей. Чтобы bower узнал о приватном репозитории, его надо задать в файле конфигурации. Для этого в папке проекта (где запускается bower) должен быть файл .bowerrc следующего содержания:

{
    "registry": "http://rnddev04.lab.croc.ru:8001"
}

Если доступ к репозиторию из сети лабы ДИТ не требуется, то можно также использовать: http://10.20.12.101:8001

Зависимости проекта хранятся в файл bower.json (в поле dependencies):

{
  "name": "Arms",
  "version": "0.1.0",
  "description": "WebClient demo-app \"ARMS\"",
  "private": true,
  "dependencies": {
    "croc.webClient": "*"
  }
}

Благодаря bower можно не хранить клиентский код WebClient в source control, а устанавливать его по необходимости. Но даже если и хранить, то обновляеть его без помощи NuGet.

Для установки всех зависимотей проекта (заданных в bower.json) надо выполнить:

bower install

Для обновления всех зависимостей проекта, заданных в bower.json, надо выполнить:

bower update

Для установки конкретного пакета ("croc.webClient"):

bower install croc.webClient

Для установки конкретного пакета конкретной версии:

bower install <package>#<version>

Добавление флага --save для bower install позволит сохранить установленную зависимость в bower.json.

По умолчанию bower устанавливает последнюю версию из Git-репозитория. Однако, ее можно задать явно в bower.json:

"dependencies": {
    "croc.webClient": "~0.11.0"
  }

В этом случае bower install установит именно эту версию.

В случае если перед версией указана тильда, будет установлена последняя дев версия пакета, если тильды нет, то только релизная. Указывая можно автоматически получать последнюю релизную версию, а ~ последнюю дев версию.
