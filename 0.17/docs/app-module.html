App-module

Весь прикладной код и код WebClient разбит на модули в терминах библиотеки RequireJS - подробней см. Module System. Но порядок выполнения кода в Require-модулях обусловлен порядком загрузки моделей с учетом их зависимостей. Т.о. код в main.js выполняющий инициализацию приложения будет выполняться после выполнения всех Require-моделей. Логично, что прикладному коду может быть удобней выполнять инициализацию после инициализации приложения. Для этого введены т.н. App-модули и их частный случай Area-модули.

Area-модуль - это колбэк, который регистрируется с помощью core.createModule. В колбэк передается объект приложения (Application). Инициализация всех App-модулей выполняется между выполнением методов preinitialize и initialize объекта Application в контексте вызова метода Application.run.

Пример:

define(["core"],
function (core) {
    "use strict";
    core.createModule("about", function (app, options) {
        areaAbout = app.areaManager.getArea("about");
        areaAbout.addState("def", {"about-history" : "history"});
        areaAbout.setDefaultState("def");
    });
});

Если инициализация относится целиком к некоторой Area, то можно определить Area-модуль, который будет инициализирован при первой активации Area. Определение Area-модуля выполняется с помощью core.createAreaModule. В метод createAreaModule передается наименование Area, для которой определяется модуль. В колбэк дополнительно к объекту приложения передается экземпляр Area.

define(["core"],
function (core) {
    "use strict";
    core.createAreaModule("about", function (app, area) {
        area.addState("def", {"about-history" : "history"});
        area.setDefaultState("def");
    });
});

Колбэк, передаваемый в core.createModule и core.createAreaModule, может возвращать объект. В этом случае Application, выполняющий инициализацию модуля, выполнит метод initialize этого объекта. Плюс при наличии поля handlers, его значение трактуется как словарь отображений наименований pub/sub событий на обработчики, которые следует подписать на эти события. Пример:

define(["core"],
function (core) {
    "use strict";
    function onAppStateChange (ev) {}
    core.createModule("breadcrumbs", function (app) {
        return {
            handlers: {
                "app.statechange": function(ev) {
                    onAppStateChange(ev);
                }
            }
        };
    });
});

В виде App-модулей оформлены многие функциональные блоки WebClient - это стандартные модули. Они располагаются в папке lib/modules. Далее каждый модуль располагается в своей папке. Идея создания модуля в том, что он добавляется в приложение одной строкой (импортом модуля) и самостоятельно меняет/расширяет поведение приложения. Импортировать стандартные модули рекомендуется в main.js.

Параметры модулей

При инициализации модуля Application проверяет наличие опций для него в объекте xconfig.modules по совпадению наименования модуля. Найденное значение передается как второй параметр колбека, переданного в createModule. Использование параметров может быть полезным для стандартных модулей.

Пример: отключение AffixManager'a. app.js

xconfig.modules.affix = {
    "disabled": true
};

lib/modules/affix/module-affix.js:

core.createModule("affix", function (app, options) {

    if (options && options.disabled)
        return;
    var affixManager = new AffixManager();
    // .. подписка на события
});
