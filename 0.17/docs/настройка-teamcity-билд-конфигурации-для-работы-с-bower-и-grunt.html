Настройка TeamCity билд конфигурации для работы с bower и grunt

Если на билд-сервере нет интернета, то nodejs, git и npm пакеты, необходимые для сборки, нужно будет установить вручную. Нужны следующие npm пакеты (зависимости не перечислены)

  • grunt
  • grunt-cli
  • bower

Доступ

С билд-сервера должен быть доступен следующий адрес: http://rnddev04.lab.croc.ru:8001 - bower репозиторий. Там находятся пакеты webClient.

Пользователь build agent'а

На билд-сервере нужно создать локального пользователя. Указать, чтобы пароль никогда не портился. Включить в группу Remote Desktop Users и временно в Administrators (удалить из роли администраторов после настройки). Дать полные права на рабочий каталог TeamCity, куда копируются исходники, и где собираются артефакты.

  1. Зайти по RDP под этим пользователем, чтобы в c:\Users для него создался каталог;
  2. Добавить в переменную PATH для данного пользователя путь c:\Users\Пользователь\AppData\Roaming\npm\;
  3. Для службы TeamCity Build Agent указать созданного пользователя. Перезапустить службу.

Дальнейшая установка пакетов и модулей должна происходить под данным пользователем, чтобы в PATH добавлялись пути со ссылкой на каталог данного пользователя.

Установка NodeJS

  1. На сайте http://nodejs.org/dist/ скачать последнюю версию msi-инсталятора NodeJS нужной битности.
  2. Установить на билд-сервере. При этом NodeJS пропишется в PATH

Установка Git

  1. На сайте https://code.google.com/p/msysgit/downloads/list?q=full+installer+official+git скачать последнюю версию Git
  2. Установить на билд-сервере.

Установить npm-модули

Предполагается, что на разработческой машине у вас уже установлены все необходимые для работы модули: grunt, grunt-cli, bower

  1. Скопировать локальный каталог c:\Users\Пользователь\AppData\Roaming\npm\ в аналогичный каталог созданного пользователя на билд-сервере. При этом будут скопированы npm-модули, установленные глобально.
  2. Скопировать содержимое каталога Client\node_modules (место локальной установки npm-модулей) на локальной машине в c:\Users\Пользователь\AppData\Roaming\npm\node_modules на билд-сервере.

Добавить шаги в build-конфигурацию

В билд-конфигурацию нужно добавить два Command Line шага для

  1. Получения vendor_components из bower-репозитория: Working directory: Client Custom script: 
    bower install
  2. Копирования клиентского кода в каталог сервера: Working directory: Client Custom script: 
    grunt build:dev

Одним шагом почему-то не получилось. webClient подтягивался, а вот копирования в серверный каталог не происходило.
