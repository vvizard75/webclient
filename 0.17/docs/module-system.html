Module System

Язык JavaScript и его реализациb для браузеров на текущий момент (ECMAScript 5) не поддерживают никаких возможностей для модульного построения приложения. Все скрипты загружаются на уровне страницы, обычно в head секции (или, что лучше, в конце body - см. тут). Причем необходимо в ручную определять порядок скриптов с учетом зависимостей между ними. Более того, чтобы компоненты, определенные в разных скриптах, работали друг с другом, необходимо модифицировать глобальное пространство имен (т.е. добавлять объекты в windows). Это все очень плохо.

Для разработки сложных приложений, особенно Single Page Application (SPA), нужна система модулей. В будущих версиях стандарта JavaScript ожидается появление нечто на эту тему. Но пока необходимо использовать библиотеки. WebClient основан на очень популярной библиотеке RequireJS.

На сайте http://requirejs.org/ подробно рассказано о проблемах, которые решает система модулей, и решениях, реализованных в RequireJS.

Разметка страницы (см. Page Layout), возвращаемая с сервера, должна содержать следующие include:

<script src="~/client/vendor/require-jquery.js" data-main="~/client/main.js" type="text/javascript"></script>

("~" будет заменен в рантайме на корневой путь от сервера).

Скрипт require-jquery.js это единственный скрипт, который надо включить на странице. В атрибуте data-main содержится наименование main-скрипта - это точка входа, по аналогии с методом main Windows-приложений.

Код main.js содержит первый вызов метода required со всеми зависимостями. Например:

require([
    "jquery", "core", "app/model-ext",
    "lib/interop/dataFacade",
    "lib/interop/backendInterop"
], function ($, core, model, DataFacade, BackendInterop) {
    $(document).ready(function () {
            // инициализация приложения (см. далее)
    });
});

Первый параметр метода require - это массив с именами модулей, от которых зависит текущий модуль. В терминах requireJS js-файл - это модуль. Имя модуля по умолчанию это имя файла без ".js", относительно директории, в которой находится main.js. Второй параметр это функция-колбэк, которая будет вызвана после загрузки всех зависимостей. Параметры этой функции - экземпляры моделей в том же порядке, как они перечисленный в первом параметре. Экземпляр модуля - это любой js-объект, который возвращает код модуля. Модуль определяется с помощью функции define. Параметры define аналогичны require - массив зависимостей данного модуля и функций, которая будет вызвана после загрузки все зависимостей. То, что вернет эта функция и будет экземпляром модуля. Имя модуля определяется его файлом. Но может быть и явно задано с помощью перегруженного варианта define с 3-мя параметрами (первым будет имя модуля, остальные такие же). Пример:

define([
    "jquery"
], function ($) {
    var core = {};
    return core;
});

RequireJS гарантирует, что каждый модуль загружается один раз (т.е. код в модуле выполняется единожды).

Используя систему модулей RequireJS мы можем писать код, который явно декларирует свои зависимости, не беспокоясь о том, когда и как они будут загружены. Более того, эти зависимости не обязательно должны быть в отдельный файлах. Специальная процедура формирования production-дистрибутива, объединяет файлы моделей, при этом код require/define остается без изменений.
