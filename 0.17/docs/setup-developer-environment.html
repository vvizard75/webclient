Setup Developer Environment

Для разработки приложений на WebClient, да и вообще для работы с front-end технологиями, требуется некоторые базовые инструменты. В данном разделе описана последовательность подготовки окружения на машине разработчика.

Chocolatey

Для установки большинства программ мы используем chocolatey:

@powershell -NoProfile -ExecutionPolicy unrestricted -Command "iex ((new-object net.webclient).DownloadString('https://chocolatey.org/install.ps1'))" && SET PATH=%PATH%;%systemdrive%\chocolatey\bin

Chocolatey - это менеджер пакетов (на основе NuGet) для Windows, аналог apt-get в Linux. Установка chocolatey - шаг абсолютно опциональный, но везде в документации подразумевается, что весь необходимый софт устанавливается с его помощью.

Если во время установки Chocolatey идет ругань о невозможности подключения к удалённому серверу, проверьте настройки proxy

Node.js
cinst nodejs.install

После установки node надо в файле конфигурации npm (node package manager) указать прокси - для этого в папке своего локального профиля Windows найдите файл .npmrc (C:\Users\UserName.npmrc) и добавьте там строки:

proxy = http://proxy.croc.ru:8000/
https-proxy = http://proxy.croc.ru:8000/

либо выполнить команды:

npm config set proxy = http://proxy.croc.ru:8000/
npm config set https-proxy = http://proxy.croc.ru:8000/
Far

Конечно можно выполнять все команды в консоле cmd, но зачем себя мучить. Используйте Far!

  • Сам Far:
cinst Far
  • Кастомная консоль ConEmu с поддержкой Far (поддерживает size окна, множество консолей в tab'ах, удобный copy/paste и много другого):
cinst ConEmu

Рекомендуется устанавливать именно в таком порядке: снача Far, потом ConEmu.

Grunt

Grunt - это task runner. Таски - это js-код, который распространяется как npm-пакеты, поддерживающие определнный API Grunt'a. С помощью тасков Grunt'a мы автоматизируем многие задачи, в частности: компиляция less, формирование дистрибутива, выполнение тестов и т.д. Grunt сам по себе это npm-модуль, который рекомендуется устанавливать локально в проекте. Но для выполнения тастов Grunt из командной строки существует модуль grunt-cli - это command-line interface для Grunt, его надо установить глобально:

npm install grunt-cli -g
Bower

Bower - это менеджер пакетов для front-end проектов. Некий аналог NPM. С помощью bower устанавливается дистрибутив WebClient (client-side only). Если вы используете NuGet и для серверной и для клиентской частей, то bower не нужен.

npm install bower -g

Подробней см. в Distribution.

Git

Git потребуется для bower, т.к. по сути bower скачивает дистрибутив из git-репозитория. Также Git потребуется для работы с основным репозиторией WebClient (http://code.rnd.croc.ru).

  1. установить Git
cinst git.install
  1. установить Git Extensions - UI-клиент Git (опционально, но рекомендуется)
cinst gitextensions
IDE

В качестве IDE для серверного проекта используется VS 2012 или 2013. Клиентский проект, если используется структура раздельных клиентского и серверного проекта (см. подробней в Project Structure), рекомендуется разрабатывать в JetBrains WebStorm.
