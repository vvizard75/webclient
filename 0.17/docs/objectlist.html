ObjectList

{toc}

Overview

ObjectList - компонент для отображения множества объектов в виде списка. Список состоит из:

  • меню (Menu)
  • фильтра (по умолчанию используется ObjectFilter)
  • представления данных

Обязательным параметром для создания списка является наименование типа объектов. Он может быть задан явно, либо опосредовано через DataSource. Для получения данных список использует источник данных - DataSource. Если источник данных для списка не задан, то он создает новый на основе типа:

dataSource = core.data.DataSource.create(app, {
        entityType: that.entityType
    });

Полученные от источника данных данные список хранит в ObservableCollection, доступной через поле items.

Для формирования UI список использует презентер (presenter). Для своего презентера список является ViewModel. Презентер может быть задан явно в опциях (см. опции presenter и Presenter), в противном случае используется презентер из опций по умолчанию (defaultOptions.Presenter). В стандартной поставке идет презентер списка SlickObjectListPresenter на основе компонента SlickGrid (дата-грид с колонками и строками). Он используется как презентер списка объектов (ObjectList), так и как презентер редактора массивного навигируемого свойства (peObjectList). Если презентер задан как класс (опция Presenter), то в конструкторе списка создается его экземпляр, в который передаются опции - объект из поля presenterOptions опций списка.

Редактируемый список

По умолчанию список содержит только достоверные данные, полученные с сервера, а все доменные объекты содержатся в изолированной UnitOfWork и не изменяются. Однако же команды "Редактировать", "Удалить" и "Создать" все-таки доступны, но по умолчанию они работают в отдельных UnitOfWork. Изменения, сделанные внутри этих команд, сначала сохраняются на сервер и лишь затем попадают в исходный список. Однако, если при создании списка задать опцию editable: true, то указанные команды будут использовать экземпляр UnitOfWork из списка. Изменения, сделанные командами, в этом режиме не сохраняются на сервер, а попадают в список и накапливаются в нем. За сохранение или откат этих изменений становится ответственен сам список; по умолчанию в нем становятся доступны команды "Сохранить" и "Отмена". Будем называть такой список редактируемым (EditableList).

Прикладные обработчики

Прикладные обработчики могут быть заданы двумя способами:

  • в объекте options, передаваемом в конструктор
  • как методы в объекте ObjectList - в самом объекте или в прототипе (т.е. классе, унаследованном от ObjectList).

Все обработчики в ObjectList вида onXXX генерируют соответствующее событие (обработчик "onXyz" генерирует событие "xyz"). Все события ObjectList описаны в статическом поле events.

Колбэки, передаваемые в объекте options в конструктор ObjectList, автоматически подписываются на события по соглашению: колбэк в поле "onInitialize" подписывается на событие "initialize" (т.е. имя поля без "on" является наименованием события).

Обработчики on* в ObjectList и события имеют одни и те же аргументы - объект args (набор полей зависит от события).

Обработчик Назначение Параметры
onDataLoading callback, вызываемый перед загрузкой данных query - объект, передаваемый в DataSource.load (и в дальнейшем в BackendInterop.load). Поле query.params - является объединением параметров фильтрации (из filter.getRestrictions) и опций экземпляра options.loadParams.
onDataPreparing callback, вызываемый после получений данных от DataSource, но до установки их в коллекцию items items - массив, объектов для установки в списке
onDataLoaded callback, вызываемый после окончания загрузки данных и установки состояния списка в loaded

Можно также задавать произвольные опции, специфичные для презентера. Они будут "проброшены" в презентер без изменения. Например, для SlickObjectListPresenter можно задать minWidth, maxWidth, resizable или любые другие опции, которые поддерживает SlickGrid.

Для сложных случае формирования значения ячейки следует использовать не форматеры, а создать свойство в объекте (или viewModel строки, см. метод createViewModel списка), в которой реализовать необходимую логику формирования значения. Если свойство viewmodel содержит html код, то для его вывода необходимо использовать "пустой" форматтер: formatter: function(val){return val;}, т.к. форматтер по умолчанию экранирует html (см. core.lang.encodeHtml). Разумеется нужно не забыть использовать эту функцию в своем свойстве.

formatter колонки

Форматер колонки можно использовать для формирования произвольного html ячейки. Иногда требуется, что внутри текста ячейки была ссыла, выполняющая команду. В этом случае форматер должен вернуть примерно следующее:

formatter: function (value) {
  return value ? "<a href='#' class='x-cmd-link' tabIndex='-1' data-cmd-name='MyCommand'>" + core.lang.encodeHtml(value.someProp()) + "</a>" : "";
}

Чтобы клик на элементе a выполнил команду, надо:

  • задать атрибут x-cmd-link
  • задать атрибут data-cmd-name с наименованием команды (сама команда может быть определена, например, в опциях списка)
  • задать атрибут tabIndex=-1, чтобы ссылка не обходилась по Tab'у (иначе с помощью клавиатуры выйти из списка будет очень сложно)

Меню и команды

Меню списка состоит из нескольких частей:

  • общее меню списка (list)
  • меню строки (row)
  • меню для выбранных строк (selection)

В общее меню списка также добавляется меню фильтра. Описания меню могут быть переданы в опциях при создании. Эти метаданные меню объединяются с метаданными дефолтных меню (возвращаемые методами createListMenuDefaults, createRowMenuDefaults и createSelectionMenuDefaults). Механизм объединения меню см. в Menu.

Меню для выбранная строк (selection) отображается если задана опция

Команды списка:

Команда Описание
Reload Перезагрузка списка
CancelReload Прерывание загрузки списка
Edit Редактирование объекта активной строки
Delete Удаление объекта активной строки

При вызове команд списка, в них передаются параметры:

  • list - для всех команд
  • object - для команд строки (меню или кастомных)

Начиная с версии 0.16, для команд могут задаваться опции в поле commandsOptions объект опций списка. В поле commandsOptions содержится объект с полями, совпадающими с именем команды (Select/Create/Edit). Значением поля является объект опций команды, либо функция, возвращающая объект опций, которая будет вызываться при каждом выполнении команды. Опции, полученные от функции, будут объеденены с опциями по умолчанию. Прототип функции:

function (args) { return {}; }

Набор опций для всех операций аналогичный:

Наименование Описание Обязательность
part Открываемый командой парт. Это может быть имя, объект парта или функция, возвращающая объекта парта (селектора/редактора/вьера и т.п.) Если не задан, то используется имя по умолчанию ("ObjectEditor:ИмяТипа" для создания/редактирования и "ObjectViewer:ИмяТипа" для просмотра, где ИмяТипа - имя типа объекта значения свойства).
openInDialog признак открытия парта в диалоге Необязательный. По умолчанию false.
freezeUrl признак "заморозки" изменения url Необязательный. По умолчанию false
partOptions объект опций для создания парта (используется если в part задано имя парта), может быть также функцией, возвращающей объект опций Необязательный.

Объект опций парта partOptions зависит от используемого класса парта и операции.

Для команд "Редактировать" (Edit) и "Просмотр" (View) поддерживается параметр navigateSiblings. Если он задан, то в редактор/viewer (парт, открываемый командой) передается список всех объектов в списке, по которым внутри редактора можно будет переключаться. Подробнее см. опция navigateSiblings в редакторе.

Фильтр

В качестве фильтра может использоваться парт, содержащий дополнительный метод getRestrictions. Если парт содержит меню (поле menu), то оно объединяется с меню списка. Метод getRestrictions должен возвращать json-объект, который будет объединяться с loadParams списка (объектом или результатом колбэка) и передаватсья в DataSource.load как поле params объекта query.

Типовым случаем является использование ObjectFilter в качестве фильтра.

Отображение ошибок валидации фильтра.

В процессе сбора ограничений фильтра в последнем могут возникнуть ошибки валидации. Стандартный фильтр на основе ObjectFilter умеет сам отображать эти нарушения (т.к. он является редактором объекта). Кастомный же фильтр, или ObjectFilter с изменённым шаблоном могут не иметь такой возможности. В подобном случае, отображение ошибок валидации берёт на себя ObjectList (ошибки отображаются в виде Hint Message). Для того, что бы ObjectList знал о том, что ему нужно отображать нарушения он ожидает в экземпляре фильтра наличие свойства canDisplayViolations с falsy значением. При использовании ObjectFilter, при необходимости можно задать это свойство с помощью опции options.canDisplayViolations.

Examples

  • список для всех объектов типа:
var list = core.ui.ObjectList.create(app, {
                entityType: "Survey"
            });
  • список на основе серверного источника данных :
var list = core.ui.ObjectList.create(app, {
                entityType: "Survey",
                dataSource: DataSource.create(app, { name: "MySurveys"})
            });
  • список с кастомным форматированием для колонок:
var list = core.ui.ObjectList.create(app, {
                dataSource: options.dataSource,
                columns: [
                    { title: 'Создан', name: 'created' },
                    { title: 'Тема', name: 'title' },
                    {
                        title: 'Организатор',
                        name: 'organizer',
                        formatter: function (value) {
                            return value ? "<i>" + core.lang.encodeHtml(value.fullName()) + "</i>" : "";
                        }
                    }
                ]
            });
  • список с фильтром (задан как наименование парта) и сортировкой, для меню списка выключена команда "Создать"
var list = core.ui.ObjectList.create(app, {
                dataSource: options.dataSource,
                autoLoad: true,
                columns: [
                    { title: 'Создан', name: 'created' },
                    { title: 'Тема', name: 'title' },
                ],
                menuList: {
                    off: ["Create"]
                },
                orderBy: ["-created", "title"],
                filter: "ObjectFilter:surveys",
                filterExpanded: true,
                hasRowNum: true
            });
  • список с кастомным форматированием для колонок: ячейка содержит ссылку, при клике на которую вызывается кастомная команда, перегружающая список с новыми условиями:
app.registerPart("ObjectList:surveys:slick-grid", function (options) {
        var list = core.ui.ObjectList.create(app, {
            uow: options.uow,
            dataSource: options.dataSource,
            autoLoad: true,
            columns: [
                { menuItem: 'Delete', width: 30 },
                { title: 'Создан', name: 'created', width: 50 },
                { title: 'Тема', name: 'title' },
                {
                    title: 'Организатор',
                    name: 'organizer',
                    formatter: function (value) {
// кастомный форматтер, ключевой момент - это класс 'x-cmd-link' для ссылки и data-атрибут data-cmd-name с наименованием команды ('OpenOrganizer'), сам команда задается в атрибуте commands опций списка
                        return value ? "<i><a href='#' class='x-cmd-link' data-cmd-name='OpenOrganizer'>" + core.lang.encodeHtml(value.fullName()) + "</a></i>" : "";
                    }
                }
            ],
            commands: {
                "OpenOrganizer": core.createCommand({
                    execute: function(args) {
                        var survey = args.object,
                            list = args.list;
                        list.filter.viewModel.organizer(survey.organizer());
                        list.reload();
                    }
                })
            },
            filter: "ObjectFilter:surveys",
            hasRowNum: true,
        });
        return list;
    });
  • список с расширеным меню для строки - добавляется новая операция и переименовывается стандартная:
var list = core.ui.ObjectList.create(app, {
    entityType: "Survey",
    menuRow: {
        update: [{
            name: "Suspend",
            title: "Приостановить голосование",
            icon: "stop",
            order: 20,
            command: {
                execute: function (args) {
