EventPublisher

В объекте Application в поле eventPublisher хранится экземпляр EventPublisher. EventPublisher - это компонент, реализующий механизм publish-subscribe - слабосвязанную публикацию и подписку событий. Компонент EventPublisher предоставляет следующее API:

/**
 * Subscribe the handler on the event.
 * @param {String} eventName Name of app event
 * @param {Function} handler Callback which will be called on the event publishing. Callback should expect argument of type AppEvent.
 * @returns {Rx.Disposable}
 */
subscribe: function (eventName, handler);

/**
 * Subscribe the handler on all events.
 * @param {Function} handler Callback which will be called on any event publishing.
 * @returns {Rx.Disposable}
 */
subscribeAll: function (handler)

/**
 * Publish event with its data.
 * @param {String} eventName Name of app event
 * @param {Object} eventData Data for the event. Subscribed callbacks get it via AppEvent.args field.
 * @param {Function} [eventData.defaultAction] Callback to be called if there're no any subscribers.
 */
publish: function (eventName, eventData)

Примеры использования:

var eventArgs = {param1: "value1"};
app.eventPublisher.subscribe("app.myevent", function (ev) {
    equal(ev.eventName, "app.myevent");
    equal(ev.args, eventArgs);
});
app.eventPublisher.publish("app.myevent", eventArgs);

Параметрами события может быть произвольный объект (либо вообще отсутствовать). Однако, обычно в качестве параметра события передается экземпляр core.SystemEvent, который обрабатывается модулем нотификаций (module-notifications). При подключении module-notifications он подписывается на все события EventPublisher'a (используя метод subscribeAll). Полученные сообщения он передает в метод NotificationManager.publishNotification.

NotificationManager трактует все объекты как нотификации, если они имеют поля message и isSystemEvent. NotificationManager поддерживает следующее Api параметра события (нотификации): | message | Текст нотификации | | html | html-разметка, используемая вместо текста | | severity | Важности события: 'error', 'warning', 'success', 'info' | | menu | метаданные меню | | defaultAction | Function - колбек, который будет выполнен при закрытии нотификации | | persist | true\false. Будет ли нотификация закрыта автоматически или потребует закрыть вручную. По умолчанию true для severity "error" и "warning" и false для остальных случаев | |
