ObjectFilter

Overview

Компонент ObjectFilter используется для задания ограничений выборки множества объектов (например, для списка). Класс ObjectFilter наследует от ObjectEditor со всеми вытекающими последствиями. В качестве viewModel как правило передается временная сущность из доменной модели (temp = true). Пример создания фильтра:

var filter = core.ui.ObjectFilter.create({
    type: "SurveyFilter", // transient entity type
    pages: [ ... ]
});

Параметры

Параметр Описание Значение по умолчанию
hideMenu скрывать меню true
comparers словарь "сравнивателей". см. ниже undefined
canDisplayViolations фильтр сам отображает ошибки валидации (подробнее см ObjectList) true

Получение ограничений

Условия фильтрации (они же ограничения) возвращаются методом getRestrictions(). Тип возвращаемого значения - простой JSON объект. В простейшем случае возвращаются все заданные в фильтре свойства. Для скалярных навигируемых свойств в качестве значения используется идентификатор объекта. Пример возвращаемого значения:

{
    "title": "foo",
    "organizer": "055e4990-fcd0-455c-9dc7-753430ae67f6"    
}

Полученный JSON-объект затем используется клиентским источником данных core.data.DataSource для передачи ограничений на сервер. Клиентский источник данных в свою очередь может работать в двух режимах:

Режим Как задать режим Способ передачи ограничений на сервер Интерпретация параметров на сервере
Возвращаются данные серверного источника данных, описанного в серверной подсистеме источников данных Используется для core.data.DataSource по умолчанию, если его наименование не совпадает ни с одной сущностью из доменной модели Ограничения передаются через URL как пары key=value. Например:\ title=foo&organizer=055e4990-fcd0-455c-9dc7-753430ae67f6 Значения URL-параметров выступают в качестве значений параметров для серверного источника данных
Возвращается набор доменных объектов с использованием механизма QueryObjects Задана опция supportQuery = true. Параметр supportQuery будет иметь значение true (если не задан явно) в случае если наименование core.data.DataSource совпадает с наименованием какой-либо сущности из доменной модели, либо наименование не задано, но задана опция entityType. Ограничения передаются через URL-параметр $filter. Значением параметра выступает строковое представление JSON-объекта, полученное с помощью метода JSON.stringify. Например:\ $filter=%7B%22title%22:%22foo%22,%22organizer%22:%22055e4990-fcd0-455c-9dc7-753430ae67f6%22%7D Значение параметра $filter используется для построения запроса с помощью механизма QueryObjects (подробнее в след. разделе)

Кастомизация формата возвращаемых ограничений

В первом режиме ограничения фильтра как правило однозначно отображаются на параметры серверного источника данных: тут все просто. Для режима QueryObject зачастую требуется более тонкое сопоставление. Например, пусть viewModel фильтра содержит свойства createdFrom и createdTo, обозначающие временные рамки даты создания опроса. Однако, в сущности Опрос, для которой и задаются ограничения фильтрации, существует одно свойство created. В этом случае на сервере ожидается значение вида:

{
    "created": {
        "ge": "YYYY-MM-DDTHH:mm:ss",
        "le": "YYYY-MM-DDTHH:mm:ss"
    }
}

Здесь "ge" и "le" - это операторы сравнения ">=" и "<=". Т.е. для режима QueryObjects значением свойства может быть не только примитив, но и некоторый объект-сравниватель (comparer). Ключи в этом объекте задают операторы сравнения, значения - эталонные значения для сравнения. Чтобы преобразовать createdFrom и createdTo к указанному виду при создании фильтра можно задать опцию comparers. Пример:

var filter = core.ui.ObjectFilter.create({
    type: "SurveyFilter",
    comparers: {
        createdFrom: function (v) {
            return { created: { ge: v } };
        },
        createdTo: function (v) {
            return { created: { le: v } };
        },
        title: "contains"
    },
    pages: [ ... ]
});

Опция comparers - это объект, наименования свойств которого совпадают со свойствами viewModel фильтра. При формировании выходных ограничений (результата метода getRestrictions()) сначала происходит поиск свойства в comparers. Если свойство найдено, то в результат подставляется значение из comparers; иначе - значение самого свойства. Значением свойства в опции comparers может быть:

  • строка, задающая оператор сравнения - подставляется объект с одним свойством. Наименование свойства - это указанная строка, значение - значение свойства viewModel. Например ограничение для title из примера выше будет выглядеть так:
{
    ...,
    title: { contains: "foo" }
}
  • функция - подставляется ее результат (см. выше пример для createdForm/createdTo)
  • объект - подставляется указанный объект; значение свойства не используется.

На сервере в данный момент поддерживаются следующие операторы сравнения

Наименование Описание
contains Вхождение строки (только для строковых свойств)
starts Начало строки (только для строковых свойств)

Добавлено в версии 0.15 | | ends | Окончание строки (только для строковых свойств) Добавлено в версии 0.15 | | eq | ==\Режим по умолчанию | | ne | != | | le | <= | | ge| >= | | lt | < | | gt | > |

Если механизма comparers недостаточно, всегда можно переопределить метод getRestrictions().
