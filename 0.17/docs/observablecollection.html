ObservableCollection

Класс ObservableCollection

По смыслу аналогичная объекту Observable, но так как является коллекцией уведомляет об изменении своего состава: добавление\удаление\изменение порядка элементов. Если содержимым коллекции являются Observable объекты, то коллекция может уведомлять об изменении любого из них событием itemChange (см. описание событий).

Методы коллекции:

Метод Сигнатура Описание
constructor function(items) создает коллекцию с элементами items (если заданы)
all function() Получение массива всех элементов
get function(i) Получение i-го элемента коллекции
count function() Получение количества элементов
indexOf function (item) Получение индекса элемента item. Если элемент не найден то -1.
add function (items) Добавление элемента (или элементов если items - массив) в коллекцию
remove function (items) Удаление элемента (или элементов если items - массив) из коллекции
move function (indexFrom, indexTo) Элементы с номерами indexFrom и indexTo меняются местами
clear function() Очистка коллекции
reset function(items) Сброс коллекции. Все элементы коллекции удаляются, затем добавляется элемент (или элементы если items массив) items.
forEach function (iterator) Итератор применяется к каждому элементу коллекции. Итерация прекращается, если итератор вернет true (либо значение приводимое к true).

Важным моментом является то, что вызов любого метода изменения коллекции генерирует только одно событие. Т.е. вызов метода reset не вызовет событие удаления для каждого старого элемента, а потом добавление для каждого нового, а только одно событие изменения коллекции.

События:

Название Аргументы Описание
get ({ prop: i, value: ret }) Вызывается каждый раз при получении элемента коллекции или свойства коллекции. i - индекс получаемого элемента, ret - значение элемента, при получении свойств коллекции (count), в prop будет содержаться название свойства
change ({ added: items, removed: oldItems }) Вызывается при изменении коллекции. added - массив добавленных элементов, removed - массив удаленных элементов. Любого из свойств объекта может не быть (если элементы не добавлялись или не удалялись). При изменении порядка элементов в коллекции (метод move) отсутствуют оба свойства.
itemChange Аналогичны аргументам события change Observable-объекта В случае если элементами коллекции являются Observable-объекты коллекция автоматически подписывается (и отписывается при удалении из коллекции) на событие change элемента, и при получении сигнала от элемента генерирует событие itemChange.

Статические методы, класса ObservableCollection:

Наименование Сигнатура Описание
isObservableCollection function (obj) Проверяет, является ли объект obj ObservableCollection. Метод вернет false, если коллекция не загружена.
