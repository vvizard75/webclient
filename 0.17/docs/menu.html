Menu

{toc}

Создание

Конструктор Menu принимает один или несколько объектов опций. Если передано несколько объектов опций, то они последовательно объединяются, при этом последние аргументы имеют больший приоритет.

var menu = new Menu(options1, options2, options3, ...);

Пример:

var menu = new Menu({
    items: [
        { name: "Edit", title: "Изменить" },
        { name: "Delete", title: "Удалить" }
    ]
}, {
    update: [
        { name: "Edit", title: "Редактировать" },
    ]
});

Созданное меню будет содержать два пункта: "Редактировать" и "Удалить".

Опции

Опции меню
Наименование Тип Описание Поведение при объединении меню
items Array Массив описаний пунктов меню Все пункты меню удаляются, вместо них устанавливаются новые.
update Array Массив описаний пунктов меню В пункты меню с соответствующими именами устанавливаются новые свойства. Новые значения свойств затирают старые. Если пункт меню с соответствующим именем не был найден, то добавляется новый.
remove Array Массив наименований пунктов меню Пункты меню с соответствующими именами удаляются.
radio Boolean Пункт меню остается активным после выбора
Опции пункта меню для items и update
Наименование Тип Описание
name String Наименование пункта меню для последующих ссылок на него. Обязательное поле
title String Заголовок, если не задан используется name
command core.command Явно заданная команда. Может быть функцией, возвращающей команду.
commandName String Наименование команды (подробнее)
html String Явно заданное html-представление пункта меню
icon String Наименование иконки (подробнее)
order Number Порядок пункта в меню
isDefaultAction Boolean Пункт меню используется по умолчанию
disabled Boolean Пункт меню недоступен
hidden Boolean Пункт меню скрыт
hideIfDisabled Boolean Скрывать пункт меню, если он недоступен
disableIfEmpty Boolean Только для пунктов с подменю. Дизаблить элемент, если подменю пустое или все его пункты недоступны
hotKey String комбинация горячих клавиш для пункта меню. подробнее ниже.
hint String Текст всплывающей подсказки (реализация зависит от презентера, но обычно используется компонент Bootstrap'a tooltip )
noFocus Boolean Опция для презентера - установить tabIndex=-1 для dom-элемнета (запретить обход по Tab)

Отображение меню зависит от презентера. Некоторые презентеры могут поддерживать не все опции.

Hotkeys

Комбинация горячих клавиш для пункта меню описывается строкой, например:

{ items: [
    { name: "MenuItem1", title: "Menu Item Title1", hotKey: "ctrl+enter" },
    { name: "MenuItem2", title: "Menu Item Title1", hotKey: "ctrl+shift+f" }
]}

Поддерживаются следующие строковые обозначения специальных клавиш:

  • "backspace", "ins", "del", "space", "tab", "enter", "esc";
  • "pageup", "pagedown", "end", "home", "left", "up", "right", "down";
  • "f1", "f2", "f3", "f4", "f5", "f6", "f7", "f8", "f9", "f10", "f11", "f12".

Кастомизация меню

Для кастомизации используются статические свойства:

  • Menu.Defaults - для кастомизации меню компонентов по умолчанию
  • Menu.Object - для кастомизации меню компонентов для заданных типов

Каждый компонент, поддерживающий меню, формирует свое меню следующим образом:

  1. берет меню по умолчанию, определенное внутри самого компонента
  2. объединяет с Menu.Default.ComponentName (if any)
  3. объединяет с Menu.Default.TypeName.ComponentName (if any)
Переопределение меню для компонента
Menu.Defaults.ReportPagePart = {
        update: [{
            name: "Print",
            title: "Печатать"
        }]
    };
Переопределение меню для отдельного типа
Menu.Object.Group = {
        RootEditor: {
            remove: [ "DetachDraft" ]
        }
    };
Наименования меню для компонентов

Компонент может использовать более одного меню для поддержки различных режимов.

ObjectList Menu.Defaults.ListRow Menu.Object.EntityType.ListRow Меню строки списка
ObjectEditor Menu.Defaults.RootEditor Menu.Object.EntityType.RootEditor Меню корневого редактора
ObjectEditor Menu.Defaults.Editor Menu.Object.EntityType.Editor Меню вложенного редактора
