Interop

Interop - слой взаимодействия приложения с сервером или другим хранилищем. Участники:

  • DataFacade
  • BackendInterop

DataFacade - это API сервера для всех клиентских компонентов. Сами вызовы сервера (ajax) реализуются другим компонентом - BackendInterop, экземпляр которого принимает DataFacade в конструкторе и хранит. DataFacade ожидает от Interop-объекта наличия методов load, save, ajax (их API совпадает с API аналогичных методов самого DataFacade).

API объекта DataFacade может расширяться модулями приложения (как стандартными, так и прикладными). В частности модуль module-security добавляет в DataFacade методы для аутентификации (login/logout/getCurrentUser) - подробно см. Security.

DataFacade и BackendIntreop реализуют следующие дополнительные (помимо ajax-вызовов) функции:

Функция Ответственный Описание
detect server update BackendIntreop в каждом ответе сервера приходит http хидер x-ver, в котором содержится версия сервера. Клиент сравнивает текущую версию с пришедшей и если она отличается генерирует событие. Такое же событие генерируется при получении события 'updateready' HTML5 объекта applicationCache. DataFacade подписывается на событие BackendIntreop и генерирует pub/sub события приложения (по умолчанию отображается как нотификация).
auto login BackendIntreop При получении 401 ошибки (Unathorized), генерируется событие, в параметрах которого передается jQuery.Deferred объект. При срабатывании Deferred запрос повторяется. Функция отключается флагом suppressAutoLogin
пробивание клиентского кеша BackendIntreop при выполнении GET-запросов по умолчанию добавляется параметр tm с текущим временем для гарантированного вызова сервера несмотря на клиентский кеш браузера. Отключается флагом suppressCacheBreakthrough
детектирование offline режима BackendIntreop на основе HTML5 событий online/offline, DataFacade подписывается на события BackendIntreop и генерирует pub/sub события приложения (по умолчанию отображается как нотификация)
pub/sub события об успешном сохранении DataFacade
pub/sub события об ошибках при вызове сервера DataFacade
уведомление о сохранении/загрузки объектов DataFacade используется для синхронизации нескольких экземпляров UnitOfWork между собой
batching TODO: объединение нескольких вызовом load/save/ajax в один http-запрос
трансформация больших GET-запросов в POST TODO: трансформация GET-запроса (load) с большим количеством параметров в POST (для обхода проблемы превышения максимальной длины URL http-запроса)

API

DataFacade API

Методы:

/**
 * Load objects.
 * @param {Object} query
 * @param {Object|String} query.source 
 * @param {String} query.source.id
 * @param {String} query.source.type
 * @param {String} query.source.propName
 * @param {String} query.preloads
 * @param {String} query.opId cancelable operation Id
 * @param {String} query.params
 * @param {Object} options same as options for ajax method
 * @return {jQuery.promise} 
 */
function load (query, options)

/**
 * Save objects.
 * @param {Array} objects domain objects in json-form (dto)
 * @param {Object} options Interop options
 * @return {jQuery.promise}
 */
function save (objects, options)

/**
 * Call server via Ajax
 * @param {Object} settings jQuery.ajax's settings
 * @param {Object} options Interop options
 * @param {Boolean} options.suppressEventOnError
 * @param {Boolean} options.suppressAutoLogin - disable auto-login on getting 401 response
 * @param {Boolean} options.suppressCacheBreakthrough - disable adding 'tm' parameter into query string
 * @return {jQuery.promise}
 */
function ajax (settings, options)

/**
 * Initiate cancellation of operation.
 * @param opId operation identifier
 */
cancel: function (opId)

События (pub/sub):

Событие Условие Параметры события
interop.error ошибка при вызове load/save severity: "important", msg: текст исключения
interop.save.success успех при вызове save severity: "success"
server_version_changed обнаружено отличие версии клиента и сервера (при выполнении любой операции) (на основе события server_version_changed от BackendInterop) version: новая версия
interop.offline Потеряна связь с сервером (на основе события offline от BackendInterop)
interop.online Восстановлена связь с сервером (на основе события online от BackendInterop)

BackendInterop API

Методы:

  • load - аналогично DataFacade
  • save - аналогично DataFacade
  • ajax - аналогично DataFacade
  • beginBatch - начало пакета, все load-запросы после вызова beginBatch будут отложены до вызова completeBatch
  • completeBatch - завершение пакета
  • checkConnection - проверка доступности сети и сервера, используется DataFacadeSmart (см. Offline)
  • bind - подписка на события "server_version_changed", "authentication_required"

События (в терминах bind/trigger):

Событие Условие Параметры события
server_version_changed изменилась версия сервера (определяется через http-заголовок x-ver), либо получено событие 'updateready' объекта applicationCache (HTML5) новая версия
authentication_required получена ошибка 401 (Unathorized) jQuery.XHR, jQuery.Deferred
online получено событие 'online' (HTML5)
offline получено событие 'offline' (HTML5)
