Part

Парт (Part) - это объект, который отображается в регионе (Region). За свое отображение отвечает сам парт: регион вызывает его метод render, передавая в него DOMElement региона.

Part API

/**
* Ask part to render itself into specified DOMElement
* @param {HTMLElement} domElement
* @return {jQuery.Deferred} promise
*/
render: function(domElement) {}

/**
* Called by Region before it shows part.
* @param {Object} [options]
* @param {String} options.reason
* @param {Object} options.state Part's state to restore (as result of applying AppState on Application start or user navigation)
*/
activating: function(options) {}

/**
* Called by Region after it shows part.
* @param {Object} [options]
* @param {String} options.reason
* @param {Object} options.state Part's state to restore (as result of applying AppState on Application start or user navigation)
*/
activated: function(options) {}

/**
* Set part's NavigationService.
* The method is being always called (if exists) by Region when it's activating a part (see Region.activatePart method).
* @param {NavigationService} navigationService
*/
setNavigationService: function(navigationService) {}

/**
* Bind part with a ViewModel
* @param {Object|Bindable} ViewModel
*/
setViewModel: function(viewModel) {}

/**
* Called by region for preparing part unloading and allowing to prevent it.
* @param {Object} options
* @return {String} reason to stay (preventing unloading) or empty
*/
queryUnload: function(options) {}

/**
* Called by region right before unloading part.
* @param {Object} options
* @return {Boolean} true if part wants to stay alive, i.e. to be deactivated only
*/
unload: function(options){}

Part CAN have field initializationTask. If so then Region.activatePart method wait for jQuery.Deferred object in the field to complete before render part.
