SlickObjectListPresenter

Параметры презентера

По умолчанию список использует SlickObjectListPresenter - презентер на основе SlickGrid. При конструировании списка через presenterOptions в него можно передать дополнительные опции для презентера:

hasCheckboxes Boolean false Добавляет в начало столбец с ролью checkbox, если он не задан явно

Роли колонок

В описании колонки для списка можно задать опцию role - так называемыю роль колонки. В зависимости от роли колонка может нести различную смысловую нагрузку и отображаться по-разному. Значением опции является строка. Возможные значения

  • data - колонка c данными. Эта роль присваивается по умолчанию.
  • command - в колонке отображается команда. Для этой роли также должна быть задана опция command. И наоборот: если для колонки не задана опция role, но задана command, то назначается роль command.
  • reorder - колокна содержит специальную иконку, с помощью которой можно перемещать строки drag&drop-ом. Если задана опция canMoveRows, то колонка с ролью reorder автоматически добавляется в начало.
  • number - в колонке отображается порядковый номер строки. Если задана опция hasRowNum, то колонка с ролью number автоматически добавляется в начало.
  • checkbox - колонка с чекбоксами для выбора строк. Если задана опция hasCheckboxes, то колонка с ролью checkbox автоматически добавляется в начало.

Роль определяет параметры отображения по умолчанию для колонки. Они задаются опцией columnRoles (внутри presenterOptions). Значением этой опции является plain объект; наименования свойств задают роли, значения - опции по умолчанию для соответствующих ролей. Пример:

var list = core.ui.ObjectList.create(app, {
                /* other options */
                columns: [
                    { role: "number", title: "№", resizable: true, width: 15, minWidth: null },
                    { title: "Наименование", name: "title", width: 50 }
                ],
                presenterOptions: {
                    columnRoles: {
                        checkbox: { resizable: true, width: 10, minWidth: null }
                    }
                }
            });

Здесь мы переопределяем опции для номеров строк (в columns наряду с другими столбцами) и чекбоксов (в presenterOptions.columnRoles).

Опции колонок

Колонки описываются для списка, а не для презентера. Однако в списке помимо стандартных можно задать произвольные опции, которые затем будут использованы презентером. SlickObjectListPresenter по описанию колонки в метаданных списка, а так же по роли колонки создает описание колонки, специфичное для SlickGrid. Все дополнительные опции из колонки копируются в колонку для SlickGrid. При этом может возникнуть ситуация, когда опции в метаданных списка конфликтуют с опциями колонок в SlickGrid. Чтобы избежать подобных конфликтов можно задавать специфичные для Slick опции в специальном поле gridOptions: свойства этого объекта будут скопированы в опции Slick-колонки (с высоким приоритетом). Например, в метаданных списка можно задать formatter для колонки (в терминах ObjectList). Однако, если мы хотим задать formatter в терминах SlickGrid, то во избежание конфликта задавать его нужно в gridOptions:

var list = core.ui.ObjectList.create(app, {
                /* other options */
                columns: [
                    {
                        name: "title",
                        gridOptions: {
                            formatter: mySlickGridFormatter
                        }
                    }
                ]
            });

Исходные метаданные колонки помещаются в поле source описания колонки для SlickGrid и могут быть использованы позднее при работе с гридом. Например:

var slickColumn = grid.getColumns()[0],
    slickFormatter = slickColumn.formatter,
    objectListColumn = slickColumn.source,
    objectListFormatter = objectListColumn.formatter;

Работа с шириной колонок

Общее описание

Итак, важным моментом в работе с гридом является управление шириной колонок. Для этой цели используются следующие опции колонки:

Наименование Значение по умолчанию Описание
width Значение опции грида defaultColumnWidth (см gridOptions Ширина колонки по умолчанию в пикселях. В случае forceFitColumns:true - относительная ширина колонки (см. ниже)
maxWidth undefined Максимальная ширина колонки
mimWidth undefined Минимальная ширина колонки
resizable true Если true, то ширина колонки может быть изменена пользователем либо гридом (см forceFitColumns в gridOptions) и объяснение ниже

Крайне важным также является параметр грида:

Наименование Значение по умолчанию Описание
forceFitColumns true Приводить суммарную ширину колонок к ширине viewport (т.е. ширине всего грида)

Параметр forceFitColumns коренным образом меняет поведение параметра width. Рассмотрим поведение грида при обоих значениях параметра:

forceFitColumns:false

В этом режиме параметр width означает фактическую ширину колонки (если width не задан, то используется defaultColumnWidth) в пикселях. Колонки располагаются одна за другой, прижимаясь к левой стороне. Если суммарная ширина колонок будет меньше чем ширина viewport, то справа от колонок останется свободное место. Если больше то в гриде появится горизонтальный скролл.

По ряду причин такое отображение грида редко используется.

forceFitColumns:true

В этом режиме грид масштабирует колонки таким образом, что бы их суммарная ширина была равна ширине viewport.

Пример: есть три колонки, со значениями ширины width:20, widht:30 и width:50. При отображении на мониторе пользователя ширина грида оказалась равна 1000px. Значит ширина первой колонки будет: 1000(20+30+50)20=200px, второй 1000/(20+30+50)30=300px, а третьей 1000/(20+30+50)*50=500px.

Есть несколько исключений из этого правила:

  1. Если колонка имеет значение параметра resizable:false, то для неё ширина widht остается абсолютной, т.е. она не будет масштабироваться. В примере выше если первая колонка будет иметь resizable:false, то её ширина так и останется 20px, ширина второй колонки будет (1000-20)/(30+50)30=368px, а третьей (1000-20)/(30+50)50=612px.
  2. Ширина колонки не может быть меньше minWidth и больше maxWidth. При масштабировании колонок значения этих параметров будут учтены

Задание ширины колонок в этом режиме можно сравнить с использованием * в wpf.

Метаданные для строк

Через опцию itemMetadata можно задать некоторые метаданные для строк. Значением опции должен быть объект метаданных в формате SlickGrid либо функция, возвращающая такой объект. Если задана функция, то в качестве аргумента ей передается элемент списка, для которого вычисляются метаданные. В качестве одного из основных применений itemMetadata видится задание css-классов для строк списка. Например, в списке пользователей мы хотим применить специальный стиль к строке, соответствующей текущему пользователю:

var list = core.ui.ObjectList.create(app, {
        entityType: "User",
        ...,
        presenterOptions: {
            itemMetadata: function (item) {
                var currentUser = item.uow && item.uow.getCurrentUser && core.lang.resolvedValue(item.uow.getCurrentUser());
                if (currentUser === item) {
                    return {
                        cssClasses: "app-current-user-row"
                    };
                }
            }
        }
    });

Допустим, мы хотим выделить текущего пользователя жирным шрифтом. Тогда в css-файле определяем следующий стиль:

.x-grid .app-current-user-row {
    font-weight: bold;
}

Опции создания грида

Объект gridOptions задает опции создания грида и передается в конструктор Slick.Grid. Формат описан здесь.

На сайте описание немного некорректное/неполное (например, указано rowHeight = 140, в реальности - 25). Полный список опций и их значений по умолчанию можно посмотреть в исходниках:) (см. начало файла client\vendor\slick\slick.grid.js).

По умолчанию SlickObjectListPresenter использует следующие значения опций:

Опция Значение
enableCellNavigation true
enableColumnReorder true
enableTextSelectionOnCells true
autoHeight true
rowHeight 32
defaultColumnWidth 100
fullWidthRows true
forceFitColumns true
multiColumnSort true
dataItemColumnValueExtractor function (item, col) { return item.get(col.id); } \

Не переопределяйте | Все значения (кроме dataItemColumnValueExtractor) можно переопределять при создании списка через presenterOptions.gridOptions.

Параметры группировки строк

Базовые параметры

SlickGrid поддерживает, с помощью специальных функций дата-адаптера, группировку (в том числе многоуровневую) строк по некоторым заданным критериям. На основе кода SlickGrid в SlickObjectListPresenter также был реализован аналогичный функционал.

Настройка группировки выполняется с помощью параметра grouping. Параметр grouping может содержать объект-описатель группировки, либо массив таких объектов, в случае использования многоуровневой группировки. Описатель группировки может иметь следующие параметры:

Параметр Значение по умолчанию Описание
getter нет Может принимать на вход одно из следующих значений:
  1. Наименование свойства в доменном объекте, соответствующем строке
  2. Функцию, принимающую доменный объект, соответствующий строке Строки группируются на основании равенства результата, возвращаемого геттером, приведенного к строке. Если геттер возвращает объект, содержащий свойство 'id' (доменный объект), то для группировки используется значение свойства 'id'. | formatter | нет | Функция, принимающая на вход описатель группы, и возвращающая html код, для заголовка группы. (описатель группы см ниже) | | comparer | function(a, b) { return (a.value == b.value?0:(a.value > b.value?1:-1));}; | Функция, принимающая на вход два описателя группы и сравнивающая их. Группы сортируются, на основании значения, возвращаемого данной функцией. (описатель группы см ниже) | | predefinedValues | пустой массив |Массив значений, аналогичных тем, что возвращает геттер. В процессе группировки, будут созданы группы соответствующие этим значениям. Таким образом, можно гарантировать, что какая либо группа всегда будет присутствовать в гриде, даже если нет соответствующих строк. | | aggregators | пустой массив | Массив аггрегаторов. (описание аггрегатора см ниже) | | aggregateEmpty | false | Показывать строку аггрегаторов в пустой группе | | aggregateCollapsed | false | Показывать строку аггрегаторов в свернутой группе | aggregateChildGroups | false | Если true, то функции аггрегатора передается массив дочерних групп, а не всех строк группы (актуально для многоуровневой группировки). Зачем это нужно не понятно, т.к. аггрегаторы, которые есть "из коробки" такой режим работы не поддерживают. | collapsed | false | Начальное положение группы (свернута\развернута) | | displayTotalsRow | true | Если false, строка аггрегатов никогда не будет показана. |

Описатель группы (поступает в качестве параметра в функции comparer и formatter):

Параметр Описание
collapsed Свернута ли группа
count Количество элементов в группе
groupingKey Ключ, по которому группируются значения (значения возвращаемого геттером, приведенное к строке либо, при наличии, значение свойства 'id' в объекте, возвращаемом геттером). При многоуровневой группировке, объединяются ключи группировки вышестоящих групп и текущей, через служебный разделитель ':|:'.
groups Массив описателей дочерних групп
level Уровень группы (при многоуровневой группировке), начиная от 0
rows Массив описателей строк группы. Доменный объект, соответствующий строке, находится в поле item описателя строки.
title Заголовок группы
totals Описатель результатов аггрегации (см ниже объект GroupTotals)
value Значение, возвращаемое геттером
Аггрегаторы

Аггрегаты в результате своей работы формируют для каждой объект GroupTotals (описание см. ниже) Из коробки поставляются четыре аггрегатора, доступных при подключении через require "plugin/gridgroup/SlickAggregators". Для включения, необходимо передать нужные аггрегаторы в массиве aggregators, например:

new SlickAggregators.Avg("someNum")

Для использования стандартных аггрегаторов, их необходимо подключить используя механизмы requrejs: "lib/ui/slick/SlickAggregators". Перечень стандартных аггрегаторов:

Конструктор Описание
new SlickAggregators.Sum("prop") Считает сумму значений свойства prop
new SlickAggregators.Min("prop") Вычисляет минимальное значение свойства prop
new SlickAggregators.Max("prop") Вычисляет максимальное значение свойства prop
new SlickAggregators.Avg("prop") Вычисляет среднее значение свойства prop (считая только те строки, где свойство prop не null, не "" и не NaN)
new SlickAggregators.Count() Считает количество строк в группе
new SlickAggregators.CountNotNull("prop") Считает количество строк в группе, где свойство prop не null, не "" и не NaN

Для вывода результата аггрегата необходимо в описании колонки задать специальный форматтер, в параметре groupTotalsFormatter. По умолчанию форматтер не задан. На вход форматтеру подаются два параметра:

Наименование Описание
totals Объект, содержащий описание результата аггрегации GroupTotals(см ниже)
columnDef Объект, содержащий описание колонки

Объект GroupTotals имеет следующие поля:

Наименование Наличие Описание
group всегда Ссылка на описатель группы
avg только если есть хотябы один аггрегатор SlickAggregators.Avg Словарь, в котором ключем является наименование колонки, а значением - результат аггрегации
min только если есть хотябы один аггрегатор SlickAggregators.Min аналогично avg
max только если есть хотябы один аггрегатор SlickAggregators.Max аналогично avg
sum только если есть хотябы один аггрегатор SlickAggregators.Sum аналогично avg
countNotNull только если есть хотябы один аггрегатор CountNotNullAggregator.Sum аналогично avg
count только если есть хотябы один аггрегатор CountNotNullAggregator.Count Количество строк в группе
другое наименование если используется кастомный аггрегатор самописные аггрегаторы могут заполнять объект GroupTotals по своему усмотрению

Таким образом groupTotalsFormatter каждой колонки получает всю информацию о результатах аггрегации в группе и в ячейке результатов аггрегации могут быть ведены не только значения по данному столбцу. Можно выводить значения нескольких аггрегатов (например сумму и среднее) или значения аггрегатов других колонок.

Пример конфигурации группировки:

presenterOptions:{
    grouping:[
        {
            getter: "task",
            collapsed: true,
            comparer: function(a, b){
                a = a.value;
                b = b.value;
                if(a.priority() != b.priority())
                    return a.priority() > b.priority() ? 1 : -1;
                if(a.deadLine() != b.deadLine())
                {
                    if(a.deadLine()==null) return 1;
                    if(b.deadLine()==null) return -1;
                    return a.deadLine() > b.deadLine() ? 1 : -1;
                }
                if(a.creationDate() != b.creationDate())
                {
                    return a.creationDate() > b.creationDate() ? 1 : -1;
                }

                return a.value.name() == b.value.name() ? 0 : a.value.name() > b.value.name() ? 1 : -1;
            },
            formatter: function(g) {
                var t = g.value;
                return (
                    "<div class='pull-right'>"
                        + "<a title='Подробная информация о задаче' href='#' tabIndex='-1' class='x-cmd-link open-task' data-cmd-name='OpenTask'><span></span></a>"
                    +"</div>"
                    +"<div class='pull-right'>"
                        + model.TaskStatus.meta.formatValue(t.status())
                    +"</div>"
                    + "<div class='pull-right task-progress'>"
                        + t.progress()
                    +"</div>"
                    + "<div class='task-title'>"
                        + "<span class='slick-group-toggle'>"
                            + t.name()
                        + "</span>"
                    + "</div>" );
            }
        }
    ]
}

Элемент с классом slick-group-toggle в заголовке группы будет сворачивать\разворачивать группу при клике на него.
