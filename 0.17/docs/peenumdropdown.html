peEnumDropDownChosen

Редактор свойства peEnumDropDownChosen построен на основе модифицированного контрола choosen. {expand: Подробней} Модификации выполнены для изменения механизма раскрытия дропдауна. Меню раскрывается в ОТНОСИТЕЛЬНОЙ позиции (относительно контейнера) и сдвигает контент страницы, а не в абсолютной (т.е. перекрывая другую разметку). Сделано это для того, что бы комбик не скрывался когда он вылезает за пределы контейнера страницы. См. инц 670079 - ui: peDropdownChosen: выпадающее меню обрезается краем редактора Изменения искать по метке CROCMOD 670079. {expand}

Использование

Класс редактора свойства peEnumDropDownChosen используется для перечисляемых не-reaonly свойств, если в метаданных (PE или свойства) задано свойство presentation="dropdown". По умолчанию используется для нефлаговых свойств.

Альтернатива (peEnumDropDownSelect2 )

Так же существует peEnumDropDownSelect2. построен с использованием библиотеки select2 (по сути является форком chosen). Подключается в качестве редактора свойства перечислимого типа при указании опции presentation="select2".

Мобильные устройства (peEnumDropDownSelect)

На мобильных устройствах peEnumDropDownChosen и peEnumDropDownSelect2 не совсем удобны для использования. Поэтому был реализован редактор свойств peEnumDropDownSelect.

Подключается в качестве редактора свойства перечислимого свойства либо при указании опции presentation="select", либо автоматически вместо peEnumDropDownChosen и peEnumDropDownSelect2 при открытии клиента на мобильном устройстве.

Представляет собой:

  • перечислимое свойство - стандартный <select />. на мобильных устройствах при клике/нажатии - открывается нативный контрол выбора;
  • флаговое свойство:
    • desktop: стандартный <select multiple />. представление может быть заменено на используемое в мобильных устройствах с помощью опции modalMultiple: true;
    • mobile: область с перечислением выбранных опций, при клике/нажатии на которую открывается модальный диалог для выбора опций в виде peEnumCheckbox
