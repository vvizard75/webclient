Domain Model

{toc}

Создание доменной модели

Доменная модель создается с помощью метода buildModel() модуля domain. В качестве параметра методу передается объект, описывающий "грязные" метаданные (raw-метаданные) модели. Пример:

var metadata = {
    entities: {
        User: {
            descr: "Пользователь",
            props: {
                firstName: { descr: "Имя", vt: "string", maxLen: 100 },
                lastName: { descr: "Фамилия", vt: "string", maxLen: 100 }
            }
        }
    }
};
var model = domain.buildModel(metadata);

Формат метаданных

Корневой объект метаданных

Корневой описатель метаданных содержит следующие свойства

Имя свойства Описание
entities Коллекция именованных сущностей
complex Коллекция именованных комплексных типов
enums Коллекция именованных перечислений

Каждое свойство-коллекция представляет из себя javascript-объект, наименования свойств которого соответствуют наименованиям объектов доменной модели. Значения свойств - описатели объектов с заданными именами.

Сущности (типы)

ИмяТипа: {
    descr: "Описание типа",
    temp: true&#124;false, // признак временного типа
    props: { /* коллекция именованных свойств */ },
    formatters: { /* коллекция именованных форматеров */ }
}

Для указания временных типов (Persistence = XTypePersistence.Transient), используется атрибут temp со значением true.

Метаданные типа также могут содержать атрибут formatters, значением которого является объект, каждое поле которого определяет форматер. Имя поля является именем форматера. Форматер с именем "default" будет использован по умолчанию, при вызове метода toString без параметров для объекта, например:

formatters: {
        "default": function() {
            // this - это текущий объект
            return this.prop1();
        }
    }

Перечисления

Метаданные перечисления:

ИмяПеречисления: {
    descr: "Описания перечисления",
    flags: true&#124;false;
    members: { /* коллекция именованных членов перечисления */ }
});

Описатель члена перечисления представляет собой объект вида:

ИмяЧлена: {
    descr: "", // отображаемое наименование
    value: целое // числовое значение
}

Комплексные типы

Наименование: {
    descr: "", // описание
    props: { /* коллекция именованных свойств */ }
}}

Формат описателей свойства комплексных типов аналогичен формату свойств обычных типов.

См. так же xfwajax:/2012/06/01/Поддержка комплексных свойств

Свойства типа

Метаданные любого свойства содержит минимум следующие атрибуты:

ИмяСвойства: {
    descr: "описание свойства",
    vt: "тип свойства"
}

Типы свойств:

Тип свойства в js-модели Тип свойства в серверной доменной модели
boolean Boolean
ui1 Byte
i2 Int16
i4 Int32
decimal Decimal
float Float
double Double
string String
text Text
date Date
time Time
dateTime DateTime

complex ???

Описание простых свойств (все кроме object и complex):

propName: {
    descr: "",
    vt: "",
    // фасеты
    // ...
    // методы
    init: // константа или функция,
    tryParse: function() {}
}
Фасеты
имя значение тип св-ва, для которого применимо Комментарий
nullable bool все
lazyLoad bool все
readOnly bool все
temp bool все значение свойства IXPropInfoValuable.IsTransient
minInclusive представление даты/времени в виде строки в формате ISO 8601 dateTime, date, time
minExclusive аналогично minInclusive dateTime, date, time
maxInclusive аналогично minInclusive dateTime, date, time
maxExclusive аналогично minInclusive dateTime, date, time

maxLen целое string, smallBin

fractionDigits целое decimal

range массив из двух элементов: левая и правая границы. значения в зависимости от типа. пример: range: \[xfwajax:24, 42\](http://wiki.rnd.croc.ru/display/xfwajax/24,+42\) все целые, все с плавающей точкой используется вместо minValue & maxValue при наличии обоих

В случае наличия у описателя свойств типов dateTime, date, time фасета IXPropInfoDateTime.DefaultValueAsCurrent - init (см. ниже) формируется генератором модели как фасет получения текущего даты/времени - "now".

О фасетах простых свойств в доменной модели - xfw3doc:Простые свойства.

Методы
  • init - установка значения по умолчанию (IXPropInfoValuable.DefaultValue). Значение соответствующего типа, фасет или функция.

В случае свойств типов dateTime, date, time в качестве начального значения используется строка в формате ISO 8601

fooProp: {
    vt: "i2",
    init: 42
},
dateTimeProp: {
    vt: "dateTime",
    init: "2001-05-11T20:22:42"
}

или

dateTimeProp: {
    descr: "dateTime Descr",
    vt: "dateTime",
    init: "now"
}
  • tryParse - метод вызывается (при его наличии) при установке значения. Во время инициализации модели генерируется методы tryParse для всех типов свойств, для которых существует парсер. Парсер получается из validation.getParser(propInfo). Подробней см. в Validation.

Описание перечисляемого свойства

При описании свойства, для значений которого требуется использовать перечисление:

  • в качестве типа свойства используется тип: "enum";
  • в атрибуте ref указывается наименование перечисления;
  • при необходимости в init задается значение по умолчанию в виде цифрового значения члена перечисления.
enumProp: {
    descr: "EnumPropDesc",
    vt: "enum",
    ref: "EnumType",
    init: 3
}

Описание навигируемых свойств

Наименование: {
    descr: "Описание",
    vt: "object",
    ref: "Имя типа-значения",
    many: true,
    opposite: "Имя обратного свойства",
    onDelete: "поведение при удалении",
    flavor: "разновидность свойства: full|aggregation|reference"
}

Атрибут opposite указывается при наличии "обратного" свойства в описании навигируемого свойства, иначе отсутствует. Атрибут many: true указывается, что свойство является множественным, т.е. образовано ролью "многие" связей один-ко-многим и многие-ко-многим.

Атрибут readOnly: true указывается при явном запрещении модификации связи. (см. атрибут ReadOnly="True"#Направлениемодификации) в описании роли). Атрибут temp: true указывается для определения временной связи (см. атрибут Transient#Прочиехарактеристики) элемента Association). Атрибут indexed: true указывается для определения индексированности роли. Атрибут orderBy: "имя свойства" - определение свойства сортировки.

Поведение связи для операции удаления описывается атрибутом onDelete, принимающем следующие значения:

Реакция Описание Формат описания в модели
Запрет Запрет действия onDelete: "restrict"
Удаление связи Удаление самой связи onDelete: "removeRelation"
Каскад Каскадное распространение действия на связанный объект(ы) onDelete: "cascade"

Действие "restrict" сейчас на клиенте не обрабатывается! Т.о., если свойство nullable, то удаление объект-значения фактически приведет к очистке ссылки (т.е. поведению "removeRelation".

Атрибут flavor может принимать одно из значений: full (по умолчанию), aggregation, reference. Значение атрибута влияет на набор операций по умолчанию в редакторе свойства:

  • reference говорит, что связь типа "ссылка", поэтому по умолчанию будут доступны операции Select (выбрать) и Unlink (удалить ссылку);
  • aggregation говорит, что связь типа "агрегация", т.е. объект-значение существует только в контексте родительского объекта, поэтому по умолчанию будут доступны операции Create (Создать), Edit (Редактировать), Delete (Удалить) (т.о. Select и Unlink исключаются)
  • "full" - доступны все операции (значение по умолчанию) Также значение flavor влияет на то, как будет открываться вложенный редактор: в случае значения "aggregation" вложенный редактор будет открываться без изменения AppState (т.е. без изменения URL браузера).

Описание комплексного свойства

При описании свойства, для значений которого требуется использовать комплексный тип:

  • в качестве типа свойства используется тип: "complex";
  • в атрибуте ref указывается наименование комплексного типа
  • прочие атрибуты аналогичны атрибутам обычных свойств
complexProp: {
    descr: "ComplexPropDesc",
    vt: "complex",
    ref: "ComplexType"
}

Общие методы

Опциональный атрибут formatter задает функцию форматер значения свойства. Если функция не задана, то форматер устанавливается дефолтный: на основании типа свойства получается функция из модуля formatters. Функции форматеры вызываются из методов получения форматированного значения: DomainObject.getFormatted и formatters.formatPropValue.

/**
* @param value
* @return {String}
*/
formatter: function(value) {}

Функция принимает значение свойства и возвращает строку.
