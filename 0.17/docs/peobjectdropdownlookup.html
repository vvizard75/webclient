peObjectDropDownLookup

Редактор скалярного навигируемого свойства с возможностями выбора из выпадающего справочника и lookup поиска (в том числе и live-search) среди значений. Во многом схож с xfw3clientdoc:XDomainObjectDropDownPropertyEditor для SmartClient. Подключается в метаданных редактора при указании соответствующего значения presentation свойства:

properties:[{
    name: "organizer",
    presentation: "dropdown"
}]
Параметры

См. также общие опции для всех Общие настройки редакторов навигируемых свойств

showEmptyItem отображать или нет в dropdown строку для выбора пустого значения. если не задано явно, то отображается для nullable свойства не задано

Представление текущего значения

По умолчанию для представления текущего значения используется formatter редактируемого свойства. Его можно заменить с помощью параметра options.formatter метаданных редактора свойства. Этот же форматтер будет использоваться для форматирования элементов dropdown-списка, если он формируется источником данных (ИД), возвращающим доменные объекты.

Загрузка данных для dropdown

Поддерживаются следующие режимы загрузки dropdown-списка (определяется опцией mode):

  • demand - при первом обращении (раскрытие комбика, lookup), режим по умолчанию;
  • preload - данные загружаются при инициализации редактора свойства;
  • live - загрузка осуществляется каждый раз при изменении строки поиска в lookup (подробнее о режиме см. ниже). Режим возможен только при использовании кастомного ИД.

Данные для отображения в dropdown загружаются с помощью DataSource. Экземпляр DataSource может быть передан в опции dataSource. Иначе он создается автоматически - используется ИД по умолчанию для типа текущего свойства. Также, наименование типа, для которого создается ИД, может быть задан явно (options.entityType).

Важно, что поведение редактора свойства и обработка опций зависят от того, является ли заданный ИД т.н. доменным ИД (т.е. возвращающим доменные объекты). ИД, создаваемый автоматически, всегда будет доменным. Но экземпляр DataSource, заданный в опциях, может быть произвольным серверным ИД. В этом случае необходимо дополнительно задать опции (один из вариантов):

  • jsonAdapter - объект с двумя методами getPresentation и getId, принимающими json-объект строки результата выполнения источника данных; результат getPresentation используется для формирования представления элемента в dropdown-списке, а getId используется для получения ObjectId объекта-значения при выборе элемента dropdown-списка.
  • displayField и idField - наименование полей в json-объекте строки результата выполнения источника данных, которые будут использованы аналогично методам getPresentation и getId; фактически создается реализация jsonAdapter:
getPresentation: function (value) {
        return value[that.options.displayField];
    },
    getId: function (value) {
        return value[that.options.idField];
    }
Lookup и Live Search

В режимах "demand" и "preload" редактор свойств позволяет производить быстрый поиск среди значений - lookup. Загруженные данные фильтруются в dropdown части по их текстовому представлению (или с помощью заданной функции), сокращая кол-во вариантов выбора. По умолчанию производится регистронезависимый поиск по текстовому представлению объекта в dropdown по условию вхождения (contains). Настройки функциональности lookup:

параметр описание значение по умолчанию
isLookup поддерживается ли в контроле режим поиска по значениям. true
lookupMinChars кол-во введенных символов, после введения которых начинается lookup 2
lookupDelay пауза в миллисекундах между последним введенным символом и стартом процесса lookup 250
lookupMatcher функция фильтрации значений при lookup, вида function(value, term). должна вернуть true, если value подходит под условия строки поиска term. в данном случае value - доменный объект.

В режиме "live" при изменении условий поиска выполняется не поиск среди загруженных значений, а перезагрузка данных датасорса с передачей ему соответствующего ограничения (параметра поиска). Соответственно lookup в данном режиме включен всегда. Параметры и отличия оных от режимов "demand" и "preload" (режим обычного lookup):

параметр описание значение по умолчанию
isLookup всегда true
lookupParam имя параметра датасорса по которому выполняется live search. Должен быть задан.
lookupMatcher аналогично обычному lookup, но в данном случае value - js объект строки результатов загрузки датасорса.
Фильтр

Имеется возможность передать датасорсу дополнительные ограничения. Для этого используется фильтр, задаваемый через параметр в метаданных options.filter или одноименное observable свойство peObjectDropDownLookup.filter.

Фильтром может являться:

  1. json объект, в котором поля являются параметрами для ИД, а значения - соответственно значениями этих параметров;
  2. любой js объект, содержащий метод toJson, возвращающий объект из п.1;
  3. функция, возвращающая что либо из п.1 и п.2. Функции передается текущая viewModel редактора свойства.

Если в options.filter или peObjectDropDownLookup.filter передан observable объект, то редактор свойства подписывается на событие "change" фильтра. При получения такого события загруженные данные считаются неактуальными и при следующем обращении (раскрытие, перезагрузка, lookup) - будут перезагружены.

Поскольку свойство peObjectDropDownLookup.filter является observable, его изменение (присвоение нового значения), так же приведет к установке флага "данные устарели".

С помощью фильтров основанных на Observable, появляется возможность связать ограничения накладываемые фильтром, с изменениями свойств viewModel (например). Подобный фильтр удобно задать в методе options.onSetViewModel (или методе onSetViewModel наследника peObjectDropDownLookup). Можно воспользоваться стандартным фильтром Filter.js ("lib/Filter"). Этот фильтр поддерживает bindable объекты в качестве значений параметров и соответственно можно с помощью байндинга связать ограничения для датасорса, например, со свойствами редактируемого объекта. Например, в метаданных редактора свойств описать следующее:

onSetViewModel: function(viewModel){
    this.filter(new Filter({
        param1: "fubar",
        param2: 21 * 2,
        param3: {ge: 5, le: 17},
        param4: binding.expr(viewModel, function () {
            return this.title() + " 42";
        })
    }));
},

При изменении ограничений накладываемых фильтром, поведением по умолчанию является сброс значения редактируемого свойства. Изменить это поведение можно задав в метаданных редактора свойств или реализовав в наследнике peObjectDropDownLookup метод onRestrictionsChanged.

Важной и полезной особенностью является то, что в этом фильтре можно использовать те же операторы сравнения, что и в фильтре списка объектов xfw3ajaxdoc:ObjectFilter#Кастомизация формата возвращаемых ограничений

Выбор значения после загрузки

Типовый случаем является выбор значения после загрузки списка. Для этого предназначены методы selectXXX, которые могут вызываться в обработчике события loaded. Пример:

{
  name: "organizer",
  presentation: "dropdown",
  onDataLoaded: function() {
    this.selectSingle.call(this);
    this.disableIfSingle.call(this);
  }
}

Другие методы:

Метод Описание
selectFirst выбор первого элемента
selectSingle выбор единственного элемента
selectIndex выбор элемента с заданным индексом
disableIfSingle редактор свойства дизейблится, если присутствует только один элемент
